Retries with backoff

import requests

from requests.adapters import HTTPAdapter

from requests.packages.urllib3.util.retry import Retry

def getRequest(url):

   s = requests.Session()

   retries = Retry(total=5, backoff_factor=1, status_forcelist=[ 429, 502, 503, 504 ])

   s.mount("https://", HTTPAdapter(max_retries=retries))

   r = s.get(url)

   print(r.status_code)

 

   return r

Threading

https://docs.python.org/3/library/threading.html

About the library

  • It’s the standard python library for threads

Under the hood

  • It’s not actually running processes at the exact same time. It still runs on a single processor, so therefore only one program can run at a time
  • However, it finds clever ways to take turns to speed up the process
  • Note: the order in which threads are run is determined by the OS. It can be pretty hard to predict.

Thread

  • The standard thread instance to start a separate thread
  • It takes in a function object (target=) and the arguments to the function in a tuple (args=)
  • Example:

import logging

import threading

import time

def thread_function(name):

   logging.info("Thread %s: starting", name)

   time.sleep(5)

   logging.info("Thread %s: finishing", name)

format = "%(asctime)s: %(message)s"

logging.basicConfig(format=format, level=logging.INFO, datefmt="%H:%M:%S")

logging.info("Main    : before creating thread")

x = threading.Thread(target=thread_function, args=(1,))

x2 = threading.Thread(target=thread_function, args=(2,))

logging.info("Main    : before running thread")

x.start()

x2.start()

logging.info("Main    : wait for the thread to finish")

logging.info("Main    : all done")

local()

  • A threading object that is specific to each individual thread which solves the problem of data being thread safe between threads
  • For example, objects like requests.Session() are not thread safe on their own. However they can be paired with local() to be thread safe
  • Note: you only need to create one threading.local() object. The object itself takes care of separating access from different threads to different data

concurrent.futures.ThreadPoolExecutor

  • A quick way to start up a group of threads
  • You tell it how many worker threads you want. It then uses a .map() to go through an iterable and pass each of those iterables to the function of a thread in the pool
  • It also does a .join() at the end of each thread in the pool; this is really nice to prevent you from forgetting to do this
  • Note: there is a confusing error in which if you pass the thread pool executor a function that takes no parameters but you pass it parameters, a hidden exception is thrown and the program terminates with no output. This makes it really hard to debug
  • Example when # threads >= # function calls:

import concurrent.futures

import logging

import time

format = "%(asctime)s: %(message)s"

logging.basicConfig(format=format, level=logging.INFO, datefmt="%H:%M:%S")

def thread_function(name):

   logging.info("Thread %s: starting", name)

   time.sleep(2)

   logging.info("Thread %s: finishing", name)

with concurrent.futures.ThreadPoolExecutor(max_workers=5) as executor: executor.map(thread_function, range(5))

Picking the number of threads

  • More threads isn’t always better. Sometimes, the extra overhead of creating and destroying threads erases the time savings you get from them
  • The correct number of threads for a program depends on the application, and is usually best found empirically

Example using threading vs. not threading[a]

  • With threading (and thread safe): Execution time ~1.5 sec (some variability)

import concurrent.futures

import requests

import threading

import time

thread_local = threading.local()

def get_session():

   if not hasattr(thread_local, "session"):

       thread_local.session = requests.Session()

   return thread_local.session

def download_site(url):

   session = get_session()

   with session.get(url) as response:

       print(f"Read {len(response.content)} from {url}")

def download_all_sites(sites):

   with concurrent.futures.ThreadPoolExecutor(max_workers=10) as executor:

       executor.map(download_site, sites)

sites = [

   "https://www.jython.org",

   "http://olympus.realpython.org/dice",

] * 80

start_time = time.time()

download_all_sites(sites)

duration = time.time() - start_time

print(f"Downloaded {len(sites)} in {duration} seconds")

Asyncio

https://docs.python.org/3/library/asyncio.html

asyncio vs. threading

  • In threading, the system knows about each thread and can interrupt it at any time to begin running in a different thread (preemptive multitasking). As such, the code doesn’t need to do anything itself to make a switch between tasks
  • Switches to other threads can happen anytime within a Python code, at any time
  • In asyncio, cooperative multitasking is used

Considerations

  • Pros
  • Speed/less memory
  • More scalable. If you increase threading from a few to hundreds there’s a noticeable slowdown. Not the case with asyncio
  • Cons
  • The code is more complicated. You have to add async and await. However it forces you to think about when a task will get swapped out, potentially leading to better code design
  • You need to use special async versions of libraries. For example the below code needs to use aiohttp instead of requests
  • This is because requests isn’t designed to notify the event loop that its blocked. Aiohttp is
  • Can lead to issues if programs misbehave (cooperative multitasking issues)

Example

  • Note: It’s super fast. ~0.75 seconds for the same example as in Threading. See https://realpython.com/python-concurrency/#asyncio-version for explanation of the code

import asyncio

import time

import aiohttp

async def download_site(session, url):

   async with session.get(url) as response:

       print("Read {0} from {1}".format(response.content_length, url))

async def download_all_sites(sites):

   async with aiohttp.ClientSession() as session:

       tasks = []

       for url in sites:

           task = asyncio.ensure_future(download_site(session, url))

           tasks.append(task)

       await asyncio.gather(*tasks, return_exceptions=True)

if __name__ == "__main__":

   sites = [

       "https://www.jython.org",

       "http://olympus.realpython.org/dice",

   ] * 80

   start_time = time.time()

   asyncio.get_event_loop().run_until_complete(download_all_sites(sites))

   duration = time.time() - start_time

   print(f"Downloaded {len(sites)} sites in {duration} seconds")

Multiprocessing

https://docs.python.org/3/library/multiprocessing.html

Under the hood

  • This module actually runs different processes at the exact same time. It’s making use of the different cores on your computer

Considerations

  • Bringing up a separate python interpreter takes more time than starting a new thread in the current Python interpreter. It’s heavyweight, and therefore sometimes it takes more resources to do this than to just run it all on multiple threads in the same interpreter

Pool

  • An object that creates separate Python interpreter processes. Each process then runs on a specified number of items in the iterable (i.e. a list)
  • Each process in the pool gets its own memory space
  • It has an initializer function parameter to hold a single session for every process. Each process will be on a different session because they have their own memory spaces

Example for an IO-bound application

  • Note: this takes about 3 seconds. Same exact program as in Threading and Asyncio

import requests

import multiprocessing

import time

session = None

def set_global_session():

   global session

   if not session:

       session = requests.Session()

def download_site(url):

   with session.get(url) as response:

       name = multiprocessing.current_process().name

       print(f"{name}:Read {len(response.content)} from {url}")

def download_all_sites(sites):

   with multiprocessing.Pool(initializer=set_global_session) as pool:

       pool.map(download_site, sites)

if __name__ == "__main__":

   sites = [

       "https://www.jython.org",

       "http://olympus.realpython.org/dice",

   ] * 80

   start_time = time.time()

   download_all_sites(sites)

   duration = time.time() - start_time

   print(f"Downloaded {len(sites)} in {duration} seconds"

Example for a CPU-bound application

  • Using multiprocessing. Runs about 2x as fast as non-concurrent version. Note that the code hardly differs from the non-concurrent version (which is very nice)

import multiprocessing

import time

def cpu_bound(number):

   return sum(i * i for i in range(number))

def find_sums(numbers):

   with multiprocessing.Pool() as pool:

       pool.map(cpu_bound, numbers)

if __name__ == "__main__":

   numbers = [5_000_000 + x for x in range(20)]

   start_time = time.time()

   find_sums(numbers)

   duration = time.time() - start_time

   print(f"Duration {duration} seconds")

  • Same program, non-concurrent. Twice the time vs. using multiprocessing

  • A way to parameterize tests within unittest using the notation with self.subTest()
  • Parameterization is a way to decrease repetition of tests with the same logic but different inputs
  • This allows us to easily test a collection of inputs
  • Each loop through the array is a test. It outputs the assertion error if the test fails

   return number * 100

# Our test class

class TestTimesTen(unittest.TestCase):

   def test_times_ten(self):

       for num in [0, 1000000, -10]: # Tests these three numbers

           with self.subTest(num):

               expected_result = num * 10

               message = 'Expected times_ten(' + str(num) + ') to return ' + str(expected_result)

               self.assertEqual(times_ten(num), expected_result, message)

Pytest

https://docs.pytest.org/en/6.2.x/contents.html

About the library

  • An easy-to-use open source unit testing framework for Python
  • It’s commonly used to write API test cases in QA automation
  • It can also be used for database and UI tests

Pytest vs. Unittest

  • Pytest allows the running of multiple tests in parallel, giving it a major performance advantage
  • Unittest is the default unit testing framework in Python
  • Implementation: unittest creates classes derived (inherited) directly from unittest.TestCase. Pytest implementation is more compact and doesn’t require creating classes; rather functions get defined and assertions are made inside
  • Therefore pytest is regarded as easier to work with because it’s simple, compact, and more efficient

Useful pytest features

  • Run multiple tests in parallel to reduce overall execution time of the suite
  • Can automatically detect the test file and test functions
  • Can skip desired subsets of tests during execution and run subsets of the test suite

Test files

  • Filenames of the format test_*.py or *_test.py
  • Running pytest without specifying a filename runs all files of the above formats in both the current directory and subdirectories
  • Other test files can be run by explicitly mentioning them

Markers

  • A way to set specific features and testing attributes to test functions
  • These markers are added via the python code
  • There are inbuilt markers and custom ones can also be made
  • Note: you can mark a test function multiple times, using multiple lines of @pytest.mark
  • Example: see @pytest.mark.<markername>

Fixtures

  • Functions that run before each test function they’re applied to
  • Used to feed data to tests (i.e. database connections, url’s, or perhaps some I/O data)
  • Note: fixture functions defined within a test file has a scope of being only within that test file
  • Fixtures available to multiple test files must be defined in a file called conftest.py
  • Example: see @pytest.fixture

Parametric tests

  • Run a test against multiple sets of inputs
  • Done in pytest by using the @pytest.mark.parametrize marker

Basic example

  • Note: my_funct() is not run because it does not start with test

dequeue

  • Use this when you need to do a lot of appending and popping but don’t need optimized index accessing
  • Container that’s similar to a list but optimized for appending and popping to the front and back, respectively.
  • The tradeoff is they don’t have optimized accessing of any index like lists do
  • Example:

from collections import deque

       # With a deque, we can append to the front directly

       bug_data.appendleft(bug)

   else:

       bug_data.append(bug) # Appends to the back

next_bug_to_fix = bug_data.popleft() # Pop from left

next_bug_to_fix = bug_data.pop() # Pop from right

UserDict

  • A wrapper container that lets us create our own dictionary
  • Such dictionaries must inherit from UserDict. Note that then you can use inherited methods like __init__(), saving the amount of code you need to write
  • We can then add additional methods or override existing ones
  • Example:

from collections import UserDict

class DisplayDict(UserDict):

   # A new custom method to increase the dictionary's functionality

   def display_info(self):

       print("Number of Keys: " + str(len(self.keys())))

       print("Keys: " + str(list(self.keys())))

       print("Number of Values: " + str(len(self.values())))

       print("Values: " + str(list(self.values())))

   # We can also overwrite a method from the dictionary class

   def clear(self):

       print("Deleting all items from the dictionary!")

       super().clear()

disp_dict = DisplayDict({'user': 'Mark', 'device': 'desktop', 'num_visits': 37})

print(disp_dict) # {'user': 'Mark', 'device': 'desktop', 'num_visits': 37}

disp_dict.display_info()

# Number of Keys: 3

# Keys: ['user', 'device', 'num_visits']

# Number of Values: 3

# Values: ['Mark', 'desktop', 37]

disp_dict.clear() # Deleting all items from the dictionary!

print(disp_dict) # {}

UserList

  • A wrapper container to create our own list
  • Same implementation as UserDict: class must inherit from UserList

from collections import UserList

class CondenseList(UserList):

   # A new custom method

   def condense(self):

       self.data = list(set(self.data))

       print(self.data)

   # Overwrite a method from List

   def clear(self):

       print("Deleting all items from the list!")

       super().clear()

condense_list = CondenseList(['t-shirt', 'jeans', 'jeans', 't-shirt', 'shoes'])

condense_list.condense() # ['shoes', 'jeans', 't-shirt']

condense_list.clear() # Deleting all items from the list!

UserString

  • A wrapper container to create our own custom string
  • Same implementation as UserDict: class must inherit from UserString
  • Example:

from collections import UserString

class IntenseString(UserString):

 

   # A new custom method for our string

   def exclaim(self):

       self.data = self.data.upper() + '!!!'

       return self.data

   # Overwrite the count method to only count a certain letter

   def count(self, sub=None, start=0, end=0):

       num = 0

       for letter in self.data:

           if letter == 'L':

               num+=1

       return num

intense_string = IntenseString("hello world i love l's")

print(intense_string.exclaim()) # HELLO WORLD I LOVE L'S!!!

print(intense_string.count()) # 5
