CSS terms

CSS

  • Cascading style sheets: add style to web pages
  • HTML is visually unappealing on its own without CSS

Rule

  • A list of CSS instructions to style an HTML element or group of elements
  • I.e. h1 { color: red; }

Property and Value

  • What part of the selector to style. color and red above, respectively

Pseudo-class selector

  • Used to style an element only when it’s being clicked
  • I.e. the : in contact-btn a:active { }

Two different CSS syntaxes

  • Ruleset (using { }’s) vs. inline style (given in the style= opening tag of an HTML element)
  • Inline styles are not recommended - requires a lot of manual work

External Stylesheet

  • Separate .css file name extensions, i.e. style.css
  • Makes the HTML file more readable
  • Use <link> in the head of the HTML file to call the .css stylesheet, i.e.

        index.html:

<head>

         <title>My Title</title>

         <link href='style.css' rel='stylesheet'>

</head>

        

        style.css:

                p {

                         color: green;

}

  • Note that you need the rel='stylesheet' attribute for it to work

Hsl

  • Color scheme of numeric values that give the mix of hue, saturation, and lightness. It makes adjusting colors easier than in RGB
  • Example: color: hsl(120, 60%, 70%);
  • Hue (first number) represents the angle on a color wheel. Red is 0 degrees, blue 240, and back to red
  • Saturation (second number) is the intensity of the color. Grayer towards 0%, richer towards 100%
  • Lightness (third number): Halfway (50%) is normal lightness. Lighter higher, dimmer lower

Basic syntax + properties

Semicolons in CSS

  • Used after the property and value inside of rule

Selector

  • Which HTML elements to style. I.e.  h1 above

Universal Selector

  • Select all elements of any type
  • Reasons you would use:
  • Reset a browser’s default style
  • Select all children of a parent element
  • Example: * { font-family: Verdana;}

Class selectors

  • Target classes of html elements within CSS
  • Example:
  • HTML

                <div class="header">

                  <h2>Heading</h2>

                   <p>Paragraph</p>

</div>

  • CSS

.header {

  color: #ffffff;

}

Multiple-class selectors

  • Add multiple css-styled classes to an HTML element, i.e. <h1 class='green bold'> Hello World! </h1>

Id selectors

  • For styling one particular page element to be styled uniquely, but all other elements of that class styled in a different way
  • Example syntax, use a #:

#{html-element-id} {

  color: #2e69a3;  

}

  • Note: id is more specific so that rule will take precedence over an HTML class

Attribute selector

  • Target all elements with a certain attribute
  • Helpful to not have to add new class or id attributes
  • General example:

[href]{

   color: magenta;

}

  • Specific example:

img[src*='winter'] {

  height: 50px;

}

  • Looks for an img element with a src attribute containing the string “winter”

Pseudo-class selector

  • Appearance of an element changes after a certain action, i.e. link is purple after you’ve clicked it or element changes after mouse hover
  • I.e. to change color of text when user hovers:

a:hover {

  color: lime;

}

<style>

  • HTML element to create an internal stylesheet
  • Must be inside the <head> element
  • Internal stylesheet example:

<head>

  <style>

    p {

      color: red;

      font-size: 20px;

    }

  </style>

</head>        

  • Note: you don’t need this if using an external stylesheet

Chaining

  • Give multiple selectors to an html element
  • I.e. a special class for html elements: h1.special { }

Selecting HTML Descendants

  • The example best demonstrates this:
  • HTML

<ul class='main-list'>

  <li> ... </li>

  <li> ... </li>

  <li> ... </li>

</ul>

  • CSS to select descendants

.main-list li { }

Multiple selectors

  • You can put multiple selectors on the same style, preventing the need to rewrite code
  • Example:

h1,

.menu {

  font-family: Georgia;

}

color

  • Styles the foreground color of element (i.e. the element text, vs. the color in background)
  • 140 named color options in CSS
  • Can also do hex or RGB colors:
  • Hexadecimal: #RRGGBB
  • RGB: rgb(255,255,255)

background-color

  • Styles an element’s background

background-image

  • The below sets a background image to anything in the .hero element:

.hero {

  background-image: url("images/myPicture.jpg");

background-size

  • Change how much of the image shows up in background
  • I.e. background-size: cover; covers the entire background while keeping the image in proportion

background-repeat

  • By default, images will repeat. Tells the compiler not to do this

background-position

  • Position the background image within the element
  • I.e. background-position: center centers it within the element

opacity

  • How transparent an element is - 0 to 1
  • Is great for captions on images
  • You can also define it in the color property via the 4th number in hsla or rgba values:
  • color: hsla(34, 100%, 50%, 0.1);
  • color: rgba(234, 45, 98, 0.33);
  • You can also say color: transparent for total opacity. It’s equivalent to rgba(0, 0, 0, 0)

!important

  • Tag that will override any other style, no matter the specificity
  • Use this sparingly or ideally, never. Often used when you have multiple stylesheets - i.e. the Bootstrap framework
  • Example:

p {

  color: blue !important;

}

Element box model + properties

Content

  • Text, images, and anything else in an HTML element

pixels

  • Note that setting pixels will be the same size on all devices - can overflow the screen on mobile

height and width

  • Size of the box of the element, in pixels. I.e. 80px
  • Note: you can also use ch for width - 1ch = the width of a 0 character
  • Scales with font-family and font-size properties

border

  • Styling of the line that surrounds the element. The property has 3 values:
  • width: in px or thin, medium, or thick
  • style: 10 styles including none, dotted, and solid
  • color

border-radius

  • In pixels or percent - percent is proportional to the width of the box
  • A 50% radius is a perfect circle

padding

  • Space between content and the border. Higher = more space around the element        
  • In px
  • Often used to make the background color area bigger. Also makes content looked less cramped within border
  • Additional specificity: padding-top, padding-right, padding-bottom, padding-left
  • Shorthand for this in clockwise rotation starting from the top:

p.my-class {

  padding: 6px 11px 4px 9px;

}

  • Shorthand for left and right sides equal. Middle value is left and right:

p.my-class {

  padding: 6px 11px 9px;

}

  • Shorthand for top and bottom equal. First value is top/bottom

p.my-class {

  padding: 6px 9px;

}

margin

  • Space between HTML page element and nearest elements
  • In px
  • Additional properties:
  • margin-top, margin-bottom, margin-left, margin-right
  • Same shorthand convention as padding
  • Note that if two divs are next to each other, the distance apart will be the sum of their two margins

margin: auto

  • Instructs the browser to adjust margins until element is centered within its containing element
  • Note: in order to center an element, a width: must be set above the margin property. Otherwise the width will automatically be set to the width of the containing element, which might not be what you want
  • For example, in the below width = 400 pixels. This is less than the width of most screens, so as long as the containing element (screen) is >= 400px the div will center within the element:

div.headline {

  width: 400px;

  margin: 0 auto;

}

Margin collapse

  • Top and bottom margins collapse while top and bottom padding does not
  • Vertical margins do not add like horizontal margins; rather the larger of the two margins sets the distance between elements on top of each other

Width and height limits

  • min-width, max-width, min-height, max-height

  • Limit the size of an element due to different screen sizes
  • This is helpful for text, for example. You don’t want it to get too small or big

overflow

  • Tells what happens to content that spills (overflows) outside of its box
  • Some typical values:
  • hidden
  • scroll: user gets a scrollbar to see remaining content
  • visible: content is displayed outside of its containing element. This is default

User agent stylesheets

  • Default stylesheets used by browsers if there’s no external stylesheet. They often have default padding and margin values
  • User agent = browser
  • To reset these to work with a clean slate: use the below CSS. This is often the first rule in an external stylesheet

* {

  margin: 0;

  padding: 0;

}

visibility

  • Hide an element from the page using visibility: hidden;
  • Note that users can still view the source code and see the element
  • Note that the space reserved for the element will still be visible on the webpage. This is different from display: none where the space is also removed

Display terms and properties

Flow

  • Browsers render CSS-less HTML elements from left to right, top to bottom
  • CSS has properties that dictate how browsers position elements

Block-level elements

  • Elements that block the full width of their parent elements so that no other elements can appear in that horizontal space
  • Being block-level is the default position of an element

position

  • How an element is configured on the page relative to others
  • 5 options:
  • static
  • Default behavior. You don’t need to put a position for the HTML element if you’re ok with the default
  • relative
  • Place an element relative to its static position (in pixels, ems, or %) in one of four ways:
  • top - down from top
  • bottom - up from bottom
  • left - to the right (away from the left side)
  • right - to the left (away from the right side)
  • Example: see green box below

.green-box {

  background-color: green;

  position: relative;

  top: 50px;

  left: 120px;

}

        

  • absolute
  • All other elements on page ignore it and act like it doesn’t exist
  • Element will be put closest to its parent element
  • You can then couple the element with offset properties to define a position
  • Note that the element will still scroll with other elements on the page
  • fixed
  • Keeps an element on the page regardless of user scrolling
  • You can then include offset properties
  • sticky
  • Keeps element in the document when scrolling, but it sticks to a certain position when the page is scrolled further
  • The included offset properties tell where the sticking happens

z-index

  • The depth of elements: deeper elements appear behind shallower ones
  • Takes in integer values depending on the order elements should be layered: i.e.

.blue-box {

  background-color: blue;

  position: relative;

  z-index: 1;

}

.green-box {

  background-color: green;

  position: relative;

  top: -170px;

  left: 170px;

}

Block vs. Inline

  • The default display value of an HTML element is either block or inline

display

  • Property that tells the type of rendering box that’s created
  • Note: position tells how we display that box on the page
  • Options:
  • block
  • Note: width can also be set for block elements
  • inline
  • inline-block
  • I.e. images. Elements can appear next to each other on the same line
  • Note: width and height can be set for inline-block elements
  • Example below. Note that if display were inline, the three sentences would be on separate lines but right on top of each other. If display were block, they would be on 3 separate lines 300px away from each other

<style>

   .rectangle {

       display: inline;

       width: 200px;

       height: 300px;

   }

</style>

<body>

   <div class="rectangle">

       <p>I’m a rectangle!</p>

   </div>

   <div class="rectangle">

       <p>So am I!</p>

   </div>

   <div class="rectangle">

       <p>Me three!</p>

   </div>

</body>

Output:

  • Elements typically in block vs. inline:
  • Common display: block elements, aka block-level elements:
  • <div>
  • <footer>
  • <h1> - <h6>
  • <header>
  • <li>
  • <main>
  • <nav>
  • <ol>
  • <p>
  • <ul>
  • <video>
  • Common display: inline elements:
  • <a>
  • <em>
  • <img>
  • <span>

display: float

  • Put elements in empty space on left or right of the parent element

display: none

  • Completely removes an item from the web page

display: flex

  • Align children of an element horizontally on a webpage
  • The arrangement now depends on how big you make the page in your browser

flex-wrap: wrap

  • Make sure no child element moves off the page when doing display: flex;

justify-content: center;

  • Center rows of child elements on a flex display

float

  • Puts an element to the left or right of a container, wrapping any other content around it
  • This is commonly used for wrapping text around an image
  • Values: left and right.
  • Must also have a width specified or else the element will take up the full width of the element that contains it
  • Note: When used for multiple elements, they can bump into each other. Particularly when they have different heights

clear

  • Instructs what to do when float elements bump into each other
  • Values:
  • left
  • Left side of the element can’t touch any other element in the parent element
  • right
  • The right side of the element can’t touch any other element within the parent element
  • both
  • Neither side of the element can touch any other element within the parent element
  • none

Typography terms + properties

Serif vs. Sans Serif

  • Serif fonts have extra details on the ends of each letter whereas Sans-Serif don’t
  • Serif fonts have been said to be easier to read. Hence books, newspapers, etc. using them
  • Sans serif has a cleaner + more modern look. On screens with lower resolution, Sans Serif can therefore look better.

Web fonts

  • Additional fonts that allow for more siles. There are limitless numbers of web fonts.
  • See Google Fonts, Adobe Fonts, for examples. You can link them to your html document.
  • You can pay for fonts at places like fonts.com
  • How to use:
  • Add the link to the font in your <head> element of html document, i.e. <link href="https://fonts.googleapis.com/css2?family=Roboto:wght@100&display=swap" rel="stylesheet">
  • Add the font to your CSS font family, i.e. p { font-family: 'Roboto', sans-serif; }

font-family

  • Property for changing font. You can have a “fallback font” too, denoted by a comma after the desired font
  • Web safe fonts are good for most browsers/operating systems
  • Best practice: if the font is multiple words, use ‘single quotes’ i.e.

h1 {

  font-family: 'Courier New';

}

 

font-size

  • Specified in one of three ways:
  • px
  • rems
  • Default font size for the web browser - tells HTML elements to scale in proportion to each other on different browsers
  • ems
  • Value that changes in proportion to font size to a parent
  • Less commonly used than px/rems

font-weight

  • Controls boldness of text. Includes:
  • bold
  • normal: if you want everything on a page to be bold except for text styled as normal
  • lighter: lighter than parent element value
  • bolder: bolder than parent element value
  • Numerical values: 1-1000. Higher is bolder. It’s common practice to use increments of 100
  • Note: Not all fonts can get a numeric font weight
  • Example:

p {

  font-weight: bold;

}

font-style

  • Used for italics. I.e. h3 { font-style: italic; }
  • You can also say normal, which is default value

text-transform

  • Style text to appear all uppercase or lowercase, i.e. h1 { text-transform: uppercase; }
  • Example use case: a website that reports breaking news might want to format all <h1> elements to be uppercase

letter-spacing

  • Set horizontal spacing between characters in an element
  • Set in px
  • Example use case: improve readability of some fonts

word-spacing

  • Set in px or em
  • Example use case: bold or large text

line-height

  • Set how tall the line containing text should be
  • Unitless number. 1.5 = 1.5x standard font height. This is preferred, since changing font size will auto adjust to this scaled up font height
  • Can also be specified in px, em, or %

text-align

  • left, right, center, or justify. Justify means to space out the text to align with the element that holds it

text-decoration

  • Values include underline for underlining text

@font-face

  • A way to add a font via download instead of hyperlink. Use for OTF, TTF, WOFF, or WOFF2 fonts
  • Download the font on google fonts or similar
  • Move the file to a folder inside your website’s working directory
  • Make the call at the top of your CSS sheet, i.e.

@font-face {

  font-family: 'MyParagraphFont';

  src: url('fonts/Roboto.woff2') format('woff2'),

       url('fonts/Roboto.woff') format('woff'),

       url('fonts/Roboto.ttf') format('truetype');

}

  • Note: font-family is a custom name. You can say anything
  • Then call the font in your stylesheet, i.e.

p {

  font-family: 'MyParagraphFont', sans-serif;

}
