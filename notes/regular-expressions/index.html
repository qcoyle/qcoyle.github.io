Terminology

Regular expression

  • A sequence of characters to be found or matched: an efficient method for processing text

Anchors

  • Ways to define a match at the start (^) and the end ($) of a string
  • Examples: ^Monkeys: my mortal enemy$ will match Monkeys: my mortal enemy but will not match Spider Monkeys: my mortal enemy in the wild or Squirrel Monkeys: my mortal enemy in the wild

Kleene star

  • A quantifier that matches the preceding character 0 or more times
  • So it’s a simpler way of doing a quantifier with { }
  • Examples: meo*w  will match me followed by 0 or more o’s followed by a w

Kleene plus

  • Matches the preceding character 1 or more times
  • Examples: meo+w will match me  followed by 1 or more o’s followed by a w

Quantifiers

  • A way to specify how many instances of a character, group, or character class must be present for a match to occur
  • Types include:
  • Fixed quantifiers
  • Optional quantifiers

Lazy vs. greedy quantifiers

  • Greedy = they take the largest option.  mo{2,4} will return the text moooo in the string moooo , not moo or mooo (even though both are technically in it)
  • Lazy = engine matches as few occurrences as possible

Optional quantifiers

  • Indicate if a character in a regex is optional - can appear 0 or 1 times
  • Denoted by ?
  • Note:
  • Examples:
  • humou?r matches both humor and humour
  • Aren't owl monkeys beautiful\? matches Aren't owl monkeys beautiful?

Optional quantifiers with grouping

  • You can have optional groupings as well
  • Examples: The monkey ate a (rotten )?banana will match The monkey ate a rotten banana  and The monkey ate a banana

Fixed quantifiers

  • A way to give the exact quantity of a character that you want to match
  • Can be done on specific characters, shorthand character classes, and range types
  • Denoted by curly braces  {}
  • Examples:
  • Match on 3 word characters: \w{3}
  • Match on  roaaar: roa{3}r
  • Match on  roaaar, roaaaar, and roaaaaar: roa{3,5}r

Grouping

  • A way to group parts of regular expressions together such that alternation is limited to just part of the regex
  • It’s denoted by an open and then a closed parentheses
  • Examples:
  • If you want to match both I love baboons and I love gorillas, you can do the regex: I love (baboons|gorillas) 

Types of text that can be matched with regular expressions

  • Literals
  • Character sets
  • Wildcards
  • Character set ranges
  • Shorthand character classes

Negated shorthand character classes

  • Matches any character that’s not in the regular shorthand classes
  • Denoted by capital expressions, i.e. \W

Shorthand character classes

  • Predefined common ranges, which make writing character set ranges much easier as it can get cumbersome to write out
  • See range types below
  • Examples:
  • \d\s\w\w\w\w\w\w\w matches a digit character, followed by a whitespace character, followed by 7 word characters. So it matches 3 monkeys

Character set ranges

  • Specify a range of characters to make a match without needing to type out every unique character
  • Examples:
  • The regex [abc], which would match any character a, b, or c, is equivalent to regex range [a-c]
  • I adopted [2-9] [b-h]ats will match the text I adopted 4 bats as well as I adopted 8 cats and even I adopted 5 hats

Range types

  • Capital letter: [A-Z]
  • Lowercase letter: [a-z]
  • Capital or lowercase: [A-Za-z]
  • Digits: [0-9]
  • Underscore: [_]
  • Single space: [ ]
  • Tab: [\t]
  • Carriage return: [\r]
  • Line break: [\n]
  • Form feed: [\f]
  • Vertical tab: [\v]
  • Shorthand character classes
  • \w = [A-Za-z0-9_] - word character
  • \d = [0-9] - digit
  • \s = [ \t\r\n\f\v] - whitespace
  • Negated shorthand character classes
  • \W = [^A-Za-z0-9_] - non word character, any character not in \w
  • \D = [^0-9]  - non digit character, any character not in \d         
  • \S = [^ \t\r\n\f\v] - non-whitespace character, any character not in  \s

Escape character

  • Denoted by \
  • Used to denote shorthand character classes
  • Used to match characters that conflict with special metacharacters (i.e. ?)

Wildcards

  • Match any character (including letters, numbers, symbols, whitespace) in a text
  • Examples:
  • ......... will completely match orangutan and marsupial
  • I ate . bananas will completely match both I ate 3 bananas and I ate 8 bananas
  • Denoted by a .
  • Note: if you want to match an actual period, use the escape character
  • Howler monkeys are really lazy\. will completely match the text Howler monkeys are really lazy.

Negated character sets

  • Allow matching of any character that is NOT stated
  • Denoted by the caret symbol ^
  • the regex [^cat] will match any character that is not c, a, or t, and would completely match each character d, o or g

Character sets

  • Allow matches of a character from a series of characters
  • This allows for matches of different or incorrect spellings
  • Note: one character set (in [ ]) only matches one character
  • Example:
  • con[sc]en[sc]us will match consensus, the correct spelling of the word, but also match the following three incorrect spellings: concensus, consencus, and concencus

Benefits of character sets over literals

  • They allow the character matching to be more flexible!

Regex Alternation

  • Match multiple types of characters using the pipe symbol |
  • Example:
  • Baboons|gorillas will provide a match for if either of these words appear in the text

Literals

  • Regular expression that matches the exact text or numbers or an exact part of the text
  • Examples:
  • Bananas matches the text “bananas”
  • Monkey can be used to match in  “The monkeys like to eat bananas”
  • 3 can match in “34”

History of regular expressions

  • Developed in 1951 by mathematician Stephen Kleene as a system to match patterns in written language with mathematical notation

Uses for regular expressions

  • See how often and where specific text pieces take place
  • Validating user input on internet to make sure it matches a predefined pattern (i.e. email addresses)
  • Examining code test results
  • Finding character patterns in text
  • Extract, edit, or delete text substrings
  • Generate reports by adding extracted strings to a collection

How regular expressions work

  • They look for a match character by character, moving left to right. If a character matches the first part of the expression

Regular expressions best practices

  • You want to make them as specific as possible as to not match unwanted parts of the text
