Introduction

HTML described in a sentence

  • Lays out the structure to content on a website, including images and videos

Markup Language

  • Computer language that defines the structure of raw text

HyperText

  • Text on a computer that gives access to other text through hyperlinks

Uniform resource locator

  • A URL!

Comments in html

  • <!-- My comment -->

HTML Terms

Tag/element/content

  • Tags are always surrounded by a < and >
  • Closing tag always has a forward slash “/” in it

Child vs. parent

  • HTML is a collection of family tree relationships
  • Child = when element inside is nested inside a parent
  • You can also have grandchildren, great-grandchildren, and more

Hierarchy

  • The relationship between elements and their children, parents, siblings, and other ancestors

Inheritance

  • Child elements can inherit behavior and styling from their parent element

Attribute

  • Content in an opening tag of an element used for providing additional information
  • Is a key value pair that contains a name and a value, i.e. <div id="intro">
  • For example, the id attribute is helpful when reusing an element

Self closing tag

  • Tag that uses a / character to close itself out
  • Used for void elements, i.e. images and links, whose structure does not require closing tags because they just contain attributes
  • The trailing slash is optional now in HTML 5

Document type declaration

  • <!DOCTYPE html>
  • Tells browsers we are using HTML
  • Must be first line of document

.index.html

  • The most commonly used name for the default page used on a website, unless the user specifies a different page
  • Other internal pages might be about.html and contact.html for “about us” and “contact us,” respectively

HTML DOM

  • 2 things:
  • The Object Model for HTML: describes the tree of html elements (which are the objects) and the properties, methods, and events for all those elements
  • An API for javascript to add/change/remove HTML elements, attributes, CSS styles, and events. Also allows JS to react to HTML events

HTML events

  • Things that happen to HTML elements, i.e.
  • A button was clicked
  • A webpage has finished loading

Semantic HTML

  • Meaning-related elements that provide details about content
  • Example: using a <header> element instead of a <div> element to contain header information
  • Reasons to use:
  • Accessibility (i.e. mobile devices, people with disabilities)
  • SEO
  • Code readability

Browser engine

  • The software in a web browser that transforms HTML, CSS, and Javascript to an interactive representation
  • Each engine renders features slightly differently due to different default styles
  • Types:
  • Blink: used by Chrome, Opera, and Edge
  • Gecko: used by Firefox
  • Webkit: used by Safari

Important Metadata Tags

<html>

  • Browser knows to interpret anything between these tags as HTML code

<head>

  • Element that contains the metadata for a web page - info about the page
  • Goes above the body tag

<title>

<link>

  • Links a CSS file to an HTML file so that the CSS can be applied
  • 3 attributes:
  • rel: relationship between current file and css file, i.e. “stylesheet”
  • type: Type of file linked
  • href: URL of file being linked

Important Body Tags

<body>

  • Tag for an element in which only it’s content inside can be displayed to the screen
  • Different types of content can be added to the body
  • The best way to ensure that all your HTML renders the same way in all browsers is to ensure that your elements remain within the opening and closing body tags

<p> vs <h*>

  • h* describes a heading, p is a paragraph
  • 6 different headings: h1 through h6
  • Functionally, not really different. But affects how search engines and browsers interpret your html

<div>

  • Generic definition of a section (aka “division) in an html document. Div’s divide the page into sections
  • No inherent visual representation. Div’s are usually styled with CSS
  • You can put anything inside the div tag
  • Allows grouping of elements to apply the same styles for any element inside

<span>

  • A way to separate small parts of content on the same line as other content
  • Usually contains html or other HTML
  • Best to use a span when you want to modify a specific piece of content that is inline with other text (i.e. changing its color). Use div for dividing content into blocks

<em>

  • Italicize

<strong>

  • Bold

<br>

  • Creates a line break in text
  • Note: it only requires a starting tag

<li>

  • List item. Is usually a child of another list element like <ul>

<ul>

  • Unordered list - bullet points
  • A list of items in no specified order

<ol>

  • Ordered list - a numbered list

<img>

  • Add image. Example: <img src="image-location.jpg">
  • Has required src attributes. Must be set to URL of image
  • Note it’s a self closing tag

<video>

  • Unlike an image tag, it requires an opening and closing tag
  • Text between the opening and closing tags will be displayed if browser can’t load the video

<a>

  • Anchor. Used for hyperlinks. Requires href attribute
  • NOTE: you can wrap pretty much any html element with an anchor element. For instance allows turning images to hyperlinks

<script>

Embed Javascript, either by containing scripting elements or pointing to an external script file via src

<section>

  • Represents a standalone section where a more specific element can’t be used
  • Typically used for document organization (i.e. a heading), whereas div is used for styling

<header> and <footer>

Important attributes

src : Source

alt: alternative text

  • A description of the image
  • Important for several reasons:
  • Gives name if image files to load
  • Helps for visually impaired users
  • Useful for SEO; helps improve your ranking
  • If the image doesn’t convey meaningful information, alt attribute should be left empty

width/height

  • For images/videos

controls

  • Attribute for videos. Tells browser to include basic video controls like pause/play/skip

href : hyperlink reference

  • For anchor elements
  • For external pages, use absolute path:  href="https://en.wikipedia.org"
  • For internal pages (in same folder), use relative path: href="./contact.html"
  • For same page: #{id name of element} href=”#{id name of element}”

target : open hyperlink in new window

  • target=”_blank” for a new window

id

  • A way to describe a unique identifier for an element
  • Cannot duplicate id names

class

  • Specifies one or multiple class names for an element
  • Usually used to point to a class in a style sheet (i.e. in the <style> element)
  • Can also be used by JavaScript to make changes to HTML elements in the class

style

  • Add inline CSS styles to html elements

Forms

<form>

  • A element that allows for submitting information
  • Important attributes:
  • action: where to send the information. I.e. "/example.html"
  • method: HTTP request type, I.e. method="POST"
  • Can also contain child elements, such as paragraphs, to give more detail

<input>

  • Element that allows accepting of user data
  • One of the most powerful and complex elements in html due to all the combinations of input types and attributes
  • Important attributes:
  • type: how it renders on webpage, what kind of data it can take, i.e. type="text"
  • Other options for type =
  • “password”
  • “number” and “step = 1” to add arrows to increase/decrease value by the step attribute
  • “range” to create a slider. Can also include “min”, “max”, “step”
  • “checkbox” with a label paired with it
  • “radio” with a “value” attribute (i.e. for a multiple choice question) and corresponding label
  • “submit” for submitting the form, with value="submit" for text on what it should say inside
  • name: required for submitting the info, i.e. name="first-text-field"
  • value: a default value for pre-filled text, if desired. I.e. value="already pre-filled"

<label>

  • Defines a label for a variety of elements
  • It also highlights the corresponding element when the label name is clicked

<select> / <option>

  • Inputs used for a dropdown menu - the option elements are children of select
  • By default, only one option button can be selected

<datalist> / <option>

  • Similar to select, but creates a text field that users can type into and filter other options
  • Must be used with an input type="text" element. Match the id’s

<textarea>

  • Bigger area to input text. Caus use the rows and cols attributes to adjust size
  • Can also click on bottom right corner to expand
  • Can also add default text as the value between the attribute tags

Validation

  • Checking user provided data against the required data
  • Types:
  • Server-side validation: another machine, typically a server, receives the info and validates
  • I.e. a login page. The form accepts it, and the server checks the key value pair
  • Client side validation:
  • Check occurs on browser, before data is sent to the server
  • HTML5 has built in client side validation
  • Benefits:
  • Helps prevent server from malicious code. So a malicious user can’t get into the server’s data
  • Quickly give user feedback, instead of having them fill in form again (if form is rejected)

Form validation attributes

  • required forces you to fill out the field. Browser will say to fill out the field
  • min=” “ and max=” “ for min/max acceptable values.
  • Throws a “your value must be greater than 1, for example, if outside range
  • minlength and maxlength for setting number of characters allowed
  • pattern to assign a regex, i.e. for a credit card number
