Terms

Javascript

  • A high level just-in-time compiled language. Web page behavior in nearly all websites and browsers is controlled by Javascript

Console

  • Panel that shows important messages (i.e. errors) - you can print to console
  • console is a keyword in javascript
  • Note: it does not write into a document (i.e. an HTML file). document.write() does that

Keyword

  • Reserved words that make up JS syntax
  • Keywords are objects
  • I.e. const

Action

  • A method
  • I.e. the .log() method

Operator

  • A character that performs a task, i.e. mathematical operators

Property

  • A characteristic of an object. Usually describes attributes
  • They are what methods get

Literal

  • Anything that represents a fixed value
  • I.e. var cat = 1, 1 is the literal. var animal = “cat”, cat is a string literal.  {name: "cat", length: 57}  is an object literal

Blocks

  • Code found inside a set of curly braces {} - used to make a list of statements
  • Known as a compound statement in other languages
  • Can be labeled or unlabeled. Example of unlabeled:

const rectangleArea = (width, height) => {

   let area = width * height;

   return area;

};

  • Used for:
  • Conditionals
  • Function declarations
  • Objects

Scope

  • Where variables can be accessed or referenced within a program
  • Is dependent on where they are defined
  • It is often thought about in relation to blocks because variables can be either outside or inside the blocks
  • Types of scope:
  • Global scope → global variables. Declared outside block
  • Block scope → variables defined inside block

Global variables vs. local variables

  • Global: Variables that can be accessed by any code in the program
  • Local: variables declared inside blocks. Only available to code that is part of the same block

Scope pollution

  • Having too many global variables can cause problems
  • Global variables get saved in the global namespace, and this can cause problems when reusing variables
  • For example, a variable num might be declared globally but then changed locally in a function. When you call  num again later, you might unknowingly call the locally declared value even though you want the global one

Scoping best practices

  • Scope your variables as tightly as possible because it will:
  1. Make your code more legible since blocks organize your code into discrete sections
  2. Code is more understandable because you know what variables are associated with what parts of program
  3. Easier to maintain your code, is more modular
  4. Saves memory in code because it doesn’t exist anymore after block executes

Module

  • Reusable pieces of code that can be exported from a program and used in another program
  • See Node.js Environment for more

React Components

  • Independent and reusable bits of code
  • They work like functions but work alone and return HTML
  • You can have Class components and Function components
  • Class Component:
  • Must include extends React.Component

class Car extends React.Component {

   render() {

       return <h2>Hi, I am a Car!</h2>;

   }

}

  • Function Component
  • Can be written with less code and are easier to understand than class components

function Car() {

   return <h2>Hi, I am a Car!</h2>;

}

  • Rendering a component:

ReactDOM.render(<Car />, document.getElementById('root'));

Typing

  • When types are checked

Strong vs weak typing

  • Implicit conversion between types is strongly-typed. Not doing so is weakly typed. Javascript is weakly typed
  • I.e. in strongly typed languages like Python and Go, "3" + 5 will raise a type error. Javascript, which is weakly typed, won’t

Run time

  • When a program is executing commands. If a program is compiled, this happens after compilation

Compiled vs. interpreted

  • When source code is translated into machine code
  • Compiled = code is translated before run time
  • Has a delay before running so that it can be translated and type checked
  • Interpreted = code is translated during execution
  • Note: compiled languages are faster because code has already been translated. No need to do this on the fly

Dynamic vs. static typing

  • In dynamic typing, types are checked during execution. For static typing, they’re checked before run-time
  • I.e. The below code runs in python because the else statement is never seen. If python was statically typed, a type error would be thrown before the code is run:

def foo(a):

   if a > 0:

       print('Hi')

   else:

       print("3" + 5)

  • Statically typed languages have better runtime performance because they won’t need to check types while running

Strict mode

  • Introduced in ECMAScript 5, it’s a way to opt in to a restricted type of Javascript that eliminates some silent errors (by returning throw errors) and can be made to run a bit faster
  • It also sometimes prevents some syntax that will eventually be deprecated
  • You can apply it to whole scripts or to specific functions
  • Exported modules (if export keyword is used) are strict by default
  • To use: say “use strict”; i.e.
  • Whole scripts:

// Whole-script strict mode syntax

'use strict';

var v = "Hi! I'm a strict mode script!";

  • Functions:

function strict() {

   // Function-level strict mode syntax

   'use strict';

   function nested() { return 'And so am I!'; }

   return "Hi!  I'm a strict mode function!  " + nested();

}

function notStrict() { return "I'm not strict."; }

Basic Syntax

  • Javascript files end in: .js
  • Print statement: console.log(helloworld); logs helloworld to console
  • It’s best practice to include semicolons at the end of every line
  • Not every statement does, but some do, so it’s good to get in the habit of it
  • Javascript is case sensitive (i.e. Console.log will not work)
  • COMMAS: Same as SQL/Java. Use // and /*  */
  • Note: you can use the latter to comment in the middle of a line. console.log(/*IGNORED!*/ 5);  // Still just prints 5 
  • Null  = null, no quotes
  • Undefined  = undefined, no quotes
  • String concatenation: console.log('hi' + name + 'ya'); // Prints 'hiya'
  • String interpolation:

const myPet = 'armadillo';

console.log(`I own a pet ${myPet}.`);

  • Get properties of an object: use the dot operator . console.log('Hello'.length); // Prints 5
  • Some common string methods:
  • .toUpperCase()
  • .startsWith()
  • .trim() removes whitespace
  • var = create a variable. var myName = 'Arya';
  • Note that let (variable can be reassigned to different values) and const (cannot be reassigned) also work to create variables
  • +, -, *, /, and % are built-in arithmetic operators
  • +=, -=, *=, and /= are mathematical assignment operators. Do math operations with less code
  • I.e. w = w + 1; → w += 1;
  • ++ and -- are increment and decrement operators. Increment/decrement by 1
  • typeof : get the data type. I.e. console.log(typeof myString); // Output: string
  • Access individual characters in a string using [ ]

const hello = 'Hello World';

console.log(hello[6]); // Returns W

Seven fundamental data types in Javascript

  • Note: 1-6 are primitive data types. Only objects are not
  1. Number: Any number including with decimals
  2. String: can be surrounded by ' ... '  or " ... " (single quotes preferred)
  3. Boolean
  4. Null (null), no quotes
  5. Undefined (undefined), no quotes
  1. Note: has a different use than null: undefined is a type, while null is an object.
  2. undefined means a variable has been declared, but no value assigned. null is an actual value you assign to variable
  3. Example:

var demo;

alert(demo); //shows undefined

alert(typeof demo); //shows undefined

var demo = null;

alert(demo); //shows null

alert(typeof demo); //shows object

  1. Symbol: unique identifiers used for more complex coding. They are a newer addition to language
  2. Object: Collections of related data

const

  • Create a read-only reference to a value
  • Note: the value created is not immutable. However, the variable identifier can’t be reassigned
  • Must be assigned a value when declared

var

  • Declares a global variable
  • Note: Since ES6, there aren’t any scenarios where var is better than let/const. So don’t use it

let

  • Declare a local variable

  • Is initialized to a value only when they are declared. So you can’t read/write them until they have been initialized
  • Optionally, you can initialize it to a value. Otherwise, it’s undefined

Temporal dead zone (TDZ)

  • The time from the start of a block (the scope) until a let variable is initialized
  • It depends on the order of execution time and not the order that the code is written. See the below examples from Mozilla docs
  • This code errors:

{ // TDZ starts at beginning of scope

   console.log(bar); // undefined

   console.log(foo); // ReferenceError

   var bar = 1;

   let foo = 2; // End of TDZ (for foo)

}

  • This code succeeds:

{

   // TDZ starts at beginning of scope

   const func = () => console.log(letVar); // OK

   // Within the TDZ letVar access throws `ReferenceError`

   let letVar = 3; // End of TDZ (for letVar)

   func(); // Called outside TDZ!

}

Hello, my name is ${userName}`);

Control Flow

If statements

let sale = true

if (sale) { // note if (sale === true) also works here

   console.log("Time to buy!")

} else {

   console.log('Wait!');

}

Semicolons

  • Required when two or more statements on same line. Optional after statements, you should NOT put one after using a closed curly bracket
  • QUIRK: Inside a () for a for loop, semicolons only go after the first and second statement. See example:

for (var i=0; i < 10; i++)  {/*actions*/}       // correct

for (var i=0; i < 10; i++;) {/*actions*/}       // SyntaxError

Comparison operators

  • >, <, >, <=, >=, ==, !=
  • Equal to and is not equal to: ===, !==
  • Note: 3 equals signs is a strict equality, meaning that the data types AND value have to be the same

77 === '77' // false (Number v. String)

77 == '77' // true

Logical operators

  • Operators that work with boolean values to add more sophistical logic to conditional statements:  &&, ||(pipe operator, OR),  ! (this last one is the bang operator, means NOT)
  • Example: if (stopLight === 'green' && pedestrians === 0)

Bang operator

  • Negates the value of a boolean

let excited = true;

console.log(!excited); // Prints false

  • Negate the value of an entire statement: use parentheses

console.log( !(false && false) );

Ternary operator

  • Denoted by ?, is a way to simplify an if else statement
  • Expressions are separated by colons :
  • If the expression evaluates to true, the first one executes. False → second one executes
  • Example: the below two statements are equivalent

let isNightTime = true;

if (isNightTime) {

   console.log('Turn on the lights!');

} else {

   console.log('Turn off the lights!');

}

isNightTime ? console.log('Turn on the lights!') : console.log('Turn off the lights!')

// Can also do something like:

isNightTime === 'yes it is' ? console.log('Turn on the lights!') : console.log('Turn off the lights!')

Truthy and falsy

  • Truthy: variables that, when used in a boolean or conditional context, evaluate to true
  • Example:

let myVariable = 'I Exist!';

if (myVariable) {

   console.log(myVariable) // Will get returned

} else {

   console.log('The variable does not exist.')

}

  • Falsy: values that include: 0, "" or '' (empty values), null, undefined, NaN
  • Example usage: Website and want to send user a personalized greeting. But sometimes people aren’t logged in and don’t have a username. A truthy statement checks for this, and sends a generic greeting message if not logged in

Short-circuit evaluation: assign variables with conditional using truthy/non truthy values:

let defaultName = username || 'Stranger'; // assigns 'Stranger' as username if not logged in (where username would be null)

Switch

  • A way to do a bunch of else if’s more efficiently
  • Note the default value to add at the end
  • Example:

let groceryItem = 'papaya';

switch (groceryItem) {

   case 'tomato':

       console.log('Tomatoes are $0.49');

       break;

   case 'lime':

       console.log('Limes are $1.49');

       break;

   case 'papaya':

       console.log('Papayas are $1.29');

       break;

   default:

       console.log('Invalid item');

       break;

}

// Prints 'Papayas are $1.29'

Arrow functions with one parameter

Hoisting

  • Functions being called before they are defined!
  • This is bad practice

Functions as first class objects

  • JavaScript functions can have properties and methods. This is not true for functions in all languages
  • So functions can have properties like .length, .name, .toString()

Functions as Data

  • Functions can behave like any other datatype. You can assign functions to variables (often for better readability), i.e.

const announceThatIAmDoingImportantWork = () => {

   console.log("I’m doing very important work!");

};

const busy = announceThatIAmDoingImportantWork;

busy(); // This function call barely takes any space!

  • busy holds a reference to the original function; points to same address in memory as announceThatIAmDoingImportantWork
  • You can invoke the name property of the function to get the actual name (note: this returns the same even if you have more references to the function)

console.log(busy.name) // Returns checkThatTwoPlusTwoEqualsFourAMillionTimes

Higher-order function

  • A function that either takes in functions as parameters, returns a function, or both
  • Callback functions: functions that get passed in as parameters and invoked in another higher-order function

Functions as Parameters

  • When a function is called by a higher-order function, you DO NOT include parentheses of the function (otherwise this would evaluate the result of the function)
  • Example: the funcParameter callback function is invoked by the timeFuncRuntime higher-order function:

const timeFuncRuntime = funcParameter => {

   let t1 = Date.now();

   funcParameter();

   let t2 = Date.now();

   return t2 - t1;

}

  • Note: an anonymous function can be a parameter too!

timeFuncRuntime(() => {

   for (let i = 10; i > 0; i--) {

       console.log(i);

   }

});

Default parameters

  • For functions, this is the ability for parameters to have a predefined value if no argument is passed into the function or if the argument is undefined

Anonymous function

  • A function with no name. You just define using function 
  • Example: function(length, with)
  • Usually used inside of another operation, i.e. a function expression

Function expression

  • A way to define a function
  • It’s best practice to use const as the keyword to define them, then assign the value an anonymous function function(var1, var2)
  • To invoke, write the name of the variable followed by parentheses with arguments passed into the function
  • They’re helpful because they don’t pollute the global scope, so the program doesn’t need to be aware of so many functions
  • Example:

// calculateArea is the function expression

const calculateArea = function (width, height) {

   const area = width * height;

   return area;

};

console.log(calculateArea(5, 4))

  • Ever since ES6 introduced arrow functions, they’re easier to declare

Arrow Functions

  • A shorter way to write functions, denoted by the () => (fat arrow).
  • This way, you don’t need to type out function every time you have to create a function
  • Example:

const rectangleArea = (width, height) => {

   let area = width * height;

   return area;

};

Do...while loop

  • Do a task once, check if while condition is met, and then keep doing it until a specific condition is no longer met

let countString = '';

let i = 0;

do {

   countString = countString + i;

   i++;

} while (i < 5);

console.log(countString); // Returns 01234

Arrays

  • Javascript’s way of making lists
  • They are pass-by-reference: When you pass an array into a function, if it’s changed inside the function that change is kept outside the function too
  • This is because you are passing a reference to the memory location of storage, so that is changing the memory

Array syntax

let myArray = ["element example", 10, true];

  • Each content item inside is called an element; elements can be different data types
  • Zero indexed

let myArray = ["element example", 10, true];

console.log(myArray[1]); // returns 10

  • Trying to log an index beyond the last element returns undefined
  • Change an element: YOU CAN DO FOR ARRAYS DEFINED BY let AND const

groceryList[1] = "avocados"; // Updates the second element

  • When you pass an array in to a function, if it’s mutated inside the function that change will be kept outside the function too (pass-by-reference)
  • Nested arrays:

const nestedArr = [[1], [2, 3]];

console.log(nestedArr[1]); // Output: [2, 3]

console.log(nestedArr[1][0]); // Output: 2

length()

console.log(myList.length); // Prints length of list

push()

myList.push("element 4", "element 5") // Adds 2 more elements to end of list

push

myList.pop(); // Removes last element from array

Iteration methods

  • Built in JavaScript array methods that allow us to manipulate elements and return values

forEach()

// myFunction is the callback function

myList.forEach(function (myFunction) {

});

// OR using arrow function:

myList.forEach(myFunction => console.log(" - " + myFunction));

// OR define the callback function first

function myFunction(element) {

};

myList.forEach(myFunction);

/*

All return each element in list

- index0

- index1

- index2

- index3

map()

  • Takes in as the callback each value in array and returns a new array

const numbers = [1, 2, 3, 4, 5];

const bigNumbers = numbers.map(number => number * 10);

console.log(bigNumbers); // [ 10, 20, 30, 40, 50 ]

console.log(numbers); // [ 1, 2, 3, 4, 5 ]

join()

console.log(secretMessage); // [ 'H', 'e', 'l', 'l', 'o', 'W', 'o', 'r', 'l', 'd' ]

console.log(secretMessage.join('')); // HelloWorld

filter()

  • Returns array of elements after filtering out specific elements from original array

const words = ['chair', 'music', 'pillow', 'brick', 'pen', 'door'];

const shortWords = words.filter(word => {

   return word.length < 6;

});

console.log(shortWords); // Returns [ 'chair', 'music', 'brick', 'pen', 'door' ]

findIndex()

  • Returns index of first element that evaluates to true
  • Returns -1 if nothing in array evaluates to callback function

const jumbledNums = [123, 25, 78, 5, 9];

  return num < 10;

reduce()

  • Returns a value after iterating through the elements of an array
  • Callback function has 2 parameters: accumulator and currentValue; accumulator starts as value of first element of array and currentValue is second. As it iterates through array, return value of function becomes accumulator and currentValue becomes next element
  • Can also take an optional second parameter to set an initial accumulator value

const newNumbers = [1, 3, 5, 7];

const myReduce = newNumbers.reduce((accumulator, currentValue) => {

   return accumulator + currentValue;

}, 2);

console.log(myReduce); // 18

Two ways to access an object’s properties

let returnProperty = (objectName, propertyName) => objectName[propertyName];

console.log(returnProperty(spaceship, "color")); // silver

Updating an object’s properties is similar to accessing them:

spaceship.["Fuel Type"] = "biofuel";

spaceship.color = "gold";

Getters

  • Methods that return internal properties of an object
  • Example:

const robot = {

   _model: '1E78V2',

   _energyLevel: 100,

       return this._energyLevel;

   }

  • Note: robot.energyLevel equals robot.getEnergyLevel in the above example. However, get lets other developers have a better understanding of what the code is doing
  • Reasons to use getters:
  • Can do something to the data when getting the property (i.e. append strings, multiply a value by 10)
  • Return different values using conditionals
  • Can use this to access calling object’s properties
  • It’s easier for developers to understand what the code is doing, instead of just having a normal function that does the same thing

       } else {

           console.log('You must assign a number to age');

       }

   }

};

person.age = 41;

console.log(person._age); // 41

  • Why use setters as opposed to just setting value directly?
  • Displays a clear intention for how the object should be used
  • Allows you to check input, i.e. the below code snippet:

   if (typeof num === "number" && num >= 0) {

       this._numOfSensors = num

   }

}

  • Why do you need a setter if you are using the _ prepending to say that the value should not be changed?
  • The _ is to say that we don’t want the property changed directly
  • It may still be wanted to be changed dynamically - under our own conditions (i.e. checked and/or modified)

Classes

Class

  • A “cookie cutter” object that can only be used to define other objects
  • Classes reduce duplicate code
  • By convention, class names are CamelCase
  • Note: no commas between class methods

class Dog {

   constructor(name) {

       this._name = name;

       this._behavior = 0;

   }

   get name() {

       return this._name;

   }

   get behavior() {

       return this._behavior;

   }

   incrementBehavior() {

       this._behavior++;

   }

}

let nikko = new Dog('Nikko'); // Create dog named Nikko

nikko.incrementBehavior(); // Add 1 to nikko instance's behavior

let bradford = new Dog('Bradford'); // Create dog name Bradford

console.log(nikko.behavior); // Logs 1 to the console

console.log(bradford.behavior); // Logs 0 to the console

Constructor

  • A special method called every time a new instance of a class is created
  • It initializes the attributes of an instance

Inheritance

  • Object oriented technique in which multiple classes share properties and methods. This decreases code complexity when done right
  • A parent class has properties/methods that child classes share
  • All parent methods are available to the child class

super()

  • For inheritance, this keyword calls the constructor of the parent class

  • Factory: Use when a variable constant is passed to the factory and constant in the returned function

Modules + Packages

Exports

  • Attach live bindings to functions, objects, or values from a module (file) so that they can be used by other programs
  • There are two types of exports: Named exports and Default exports
  • export cannot be used in embedded scripts

Named export

  • Provide multiple exports per file. These named exports are then imported using the import keyword and curly braces syntax
  • You can rename exports

export const myComponent = () => {}

export function myFunction(){}

// Alternate syntax

export { myFunction as function1,

   myVariable as variable };

Default exports

  • Allows only one export (i.e. a single variable, funcion, or object) per file
  • If you have multiple default exports, the last one is what overrides
  • Part of ES6
  • You cannot use it with Node.js
  • A default export can be imported with any name

// file main.js

export default function () {}

// Another file

import funct from './main.js';

Named imports

  • Import specific values only

import { specialty, isVegetarian } from './menu'; // Note: menu.js is the filename

console.log(specialty);

import

  • Use to import live bindings exported by another modules
  • Recall that imported modules are in strict mode by default
  • Cannot be used in embedded scripts unless the script has type="module"
  • Syntax from docs:

import defaultExport from "module-name";

import * as name from "module-name";

import { export1 } from "module-name";

import { export1 as alias1 } from "module-name";

import { export1 , export2 } from "module-name";

import { export1 , export2 as alias2 , [...] } from "module-name";

import defaultExport, { export1 [ , [...] ] } from "module-name";

import defaultExport, * as name from "module-name";

import "module-name";

var promise = import("module-name");

import()

  • A dynamic import statement. It does not require scripts to have type="module"

  • Note that import/export only work via HTTP(s), not locally. Use a static server or live server to test them instead

Javascript vs. Python

Real world differences

  • Front-end usability

  • While both can be used for the front end, JavaScript is much more widely used. This is because it’s universal and it’s easy to use. It’s the only programming language natively supported by web browsers

  • Back-end usability

  • Both Python and Node.js are two of the most used and back-end languages

  • Modules

  • Python has lots of modules. Everything included
  • JS has very few modules. Only some like date, math, regexp, and JSON. It gets most of its functionality through the browser environment

  • Runtime

  • Python is an interpreted language
  • Javascript is an interpreted language, but when using node Google’s V8 engine compiles JS to native machine code before executing. This increases performance. So it seems to depend on how you execute it

  • Run speed (performance)

  • JS is much faster than python. Python has a single code flow, which can make it take longer to process requests

  • Maintenance

  • Python is an easier language to maintain than JS because it is better designed. It emphasizes code readability and maintainability but it’s more expressive

  • Scalability

  • Python does not support multithreading.
  • JS (using Node.js) uses different modules and can work asynchronously for multithreading. This makes it more scalable. It also has had more frameworks developed as it becomes more popular
  • Type checking
  • Both are dynamically typed - they check types at runtime. Therefore you don’t need to declare a variable’s type
  • Note: for bigger programs this can become an issue for maintenance as it’s less secure. It also makes it harder for people and editors alike to understand the code

  • Cross-stage

  • Both languages are cross-stage: a developer can create one app that works on MacOS, Linux, and Windows

  • Data science

  • With the packages available and its native support of more data types, python is much more widely used for data science

Syntax differences

  • Numeric data types

  • Python has 3: int, float, and complex
  • JS has two: Number (includes integers + floating point values, theres no int in JS) and BigInt

  • Primitive data types

  • Python has 4: int, float, bool, and str
  • JS has six: undefined, Boolean, String, Number, BigInt, and Symbol

  • None vs. null

  • Python uses None to indicate no value for a variable
  • JS use null instead

  • undefined

  • Python has no such thing. You must assign an initial value.
  • JS, you can declare a variable without assigning an initial value. Its value is undefined

  • Built-in data structures - tuples

  • Python: it’s like a list but it’s immutable. Use it to store data that shouldn’t be modified
  • JS: No built-in data structure that does this. You can make a similar structure if you want

  • Built-in data structures - Lists vs. arrays

  • Python: Lists store a sequence of values. Can be modified and manipulated
  • JS: A Javascript array is the same thing as a python list

  • Built-in data structures - Dictionary (aka hash tables)

  • Python: This is a built-in data structure where key value pairs can be created
  • JS: No built in data structure for hash tables

  • Logical Operators

  • Python: and, or, and not
  • JS: &&, ||, and !

  • User input

  • Python: input = input(“Enter a value”). Displayed in console
  • JS: input = window.prompt(“Enter a value”) will be displayed on the browser

  • Switch/case

  • Python: Does not have this type of control flow
  • Javascript: Switch

  • For loops

  • Python: can use for i in range(n): or for iterables, for elem in iterable:
  • JS: need to explicitly give start value, end value, and iterator. For iterables, you can do for (var elem of iterable) { } and for objects, can do for (const property in object) { console.log(property, object[property]);

  • Python: cannot do this
  • JS: see Do...while loop

  • Python: yes, using class
  • JS: yes, using class

  • JS: See Constructor
