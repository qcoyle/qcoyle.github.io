Terminology

What is Vue

  • A flexible model-view-viewmodel front end JS framework used for app and web development
  • Uses a virtual DOM
  • Was originally designed for single-page apps and user interfaces
  • Has been growing in use since 2017

Pros and cons of Vue

  • Pro’s
  • Lower learning curve due to user-friendliness
  • Some refer to it as an extension of Angular and React, while also providing the ability to manage HTML that’s already rendered by the server (asynchronous server-side rendering)
  • Incredibly lightweight
  • 6th most-used front end framework with >200% growth in recent years
  • Used in industry: Netflix uses it for streaming. Facebook also uses it on some projects
  • Con’s
  • Not as suitable for large projects
  • Smaller community and not as many developers. Not financially supported by larger companies
  • Language barrier due to lots of discussion about it being in Chinese. Is popular with Alibaba/Xiaomi
  • Fewer plugins than Angular or React; for large projects you can’t fully rely on Vue
  • Difficulties with older versions of iOS/safari
  • High level of code flexibility leads to more errors in code, which can cause delays

When to use Vue

  • Adding interactivity to your project - easy compatibility with Express, Django, etc.
  • Adding improvements to your UI
  • Prototype development

Choosing a web framework

  • jQuery: Seems to be solving a problem that doesn’t exist anymore: normalizing behavior across different browsers
  • React: Based on component based architecture (CBA)
  • Angular: Based on MVC design pattern. Steeper learning curve

Directive

  • A token in the markup that tells Vue to do something to a DOM element
  • Example: <div v-text="message"></div>

Component

Props

  • Custom attributes that can be registered to a component to become a property on the component instance

Inline styling with front end frameworks

  • While inline styling with raw HTML makes code harder to read and maintain, using it with front end frameworks can do the opposite. Allows for CSS to be limited to small, reusable pieces of front end code

General Syntax

NOTE: This doc is in v2 of Vue. v3 has now been released and contains some significant changes. Especially for installation. See https://v3.vuejs.org/guide/introduction.html

Adding Vue to a project

  • Add the following script to your HTML. Note that the defer attribute is used to load the webpage before Vue

<!-- index.html -->

<head>

   <script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js" defer></script>

   <script src="./js/app.js" defer></script>

 </head>

  • Note that the app.js script must be imported after Vue gets imported
  • Create a new instance of Vue(). The Vue constructor takes only one object argument

// app.js

const app = new Vue({});

Options object

  • The argument taken in by the Vue constructor
  • Arguments needed by Vue are added as key value pairs

el

  • Stands for el: use to specify which portion of the HTML gets access to Vue
  • Takes in a CSS selector to target a particular section of the HTML. This could refer to a div with id=app

// app.js

const app = new Vue({

   el: '#app'

 });

Templating

  • Use two sets of curly brackets to access values in the data object

<div id="app">

   <h2>Hello, {{ username }}</h2>

</div>

Create a component

const Tweet = Vue.component('tweet', {

   props: ['message', 'author'],

   template: '<div class="tweet"><h3>{{ author }}</h3><p>{{ message }}</p></div>'

});

data

  • Key to specify an object of key value pairs of dynamic data to be displayed by the app
  • In most apps, this data would come from a database - which might pull on the username or user id
  • Example:

const app = new Vue({

   el: '#app',

   data: {

       username: 'Jane'

   }

});

computed

  • A way to store dynamic data that’s computed using other pieces of dynamic data
  • Use cases include calculating the date, the season, the time, or conditionals on other parameters. It's also a helpful way to combine multiple {{ }} terms right next to each other into one variable, i.e. a fullName variable. Example:

const app = new Vue({

   el: '#app',

   data: {

       age: 65

   },

   computed: {

       type: function() {

           if (this.age < 18) {

               return "Child";

           } else if (this.age < 35) {

               return "Young adult";

           } else if (this.age < 65) {

               return "Adult";

           } else {

               return "Senior";

           }

       }

   }

computed: get and set

  • Computed property terms that allow values to be re calculated in response to inputs - all in the front end
  • Computed values only recompute when a dynamic value used inside the getter function changes
  • Example:

const app = new Vue({

   el: '#app',

   data: {

       age: "",

       submitted: false

   },

   computed: {

       ageLevel: {

           get: function() {

               if (this.age >= 0) {

                   if (this.age < 18) {

                       return "Child";

                   } else if (this.age < 35) {

                       return "Young adult";

                   } else if (this.age < 65) {

                       return "Adult";

                   } else {

                       return "Senior";

                   }

               }

           },

           set: function(newAgeLevel) {

               if (newAgeLevel === "Child") {

                   this.age = 0;

               } else if (newAgeLevel === "Young adult") {

                   this.age = 18;

               } else if (newAgeLevel === "Adult") {

                   this.age = 35;

               } else if (newAgeLevel === "Senior") {

                   this.age = 65;

               }

           }

       }

   }

});

<span>Change Level:</span>

       <select v-model="ageLevel">

           <option>Child</option>

           <option>Young adult</option>

           <option>Adult</option>

           <option>Senior</option>

       </select>

       <p v-if="!isNaN(parseInt(age))">You are at least {{ age }} years old. You are a {{ ageLevel }}.</p>

watch

  • A way to make update existing app properties when dynamic changes occur
  • The watch object contains all the properties to watch
  • Note: computed values update more efficiently than watch. Use computed when you can
  • Example. Notice that it doesn’t create a new dynamic property. It just updates existing ones

const app = new Vue({

   el: '#app',

   data: {

     currentLanguage: 'Spanish',

     supportedLanguages: ['Spanish', 'Italian', 'Arabic'],

     hoursStudied: 274

   },

   watch: {

     currentLanguage: function (newCurrentLanguage, oldCurrentLanguage) {

       if (supportedLanguages.includes(newCurrentLanguage)) {

         this.hoursStudied = 0;

       } else {

         this.currentLanguage = oldCurrentLanguage;

       }

     }

   }

 });

methods

  • Where functions used in the program are stored - i.e. helper functions or event handlers
  • These methods can then be added to an element (i.e. a <button>) to run when necessary

const app = new Vue({

   el: "#app",

   data: {

       hoursStudied: 300

   },

   methods: {

       resetProgress: function () {

           this.hoursStudied = 0;

       }

   }

});

<button v-on:click="resetProgress">Reset Progress</button>

Directives Syntax

v-if

  • A directive that only displays HTML is the statement returns to true
  • In the below example, it checks the .data value to see if the user is logged in

<button v-if="userIsLoggedIn">Log Out</button>

<button v-if="!userIsLoggedIn">Log In</button>

v-for

  • A way to render an array of items in the same way
  • The below code templates through every item in the todoList array in .data

<ul>

   <li v-for="todo in todoList">{{ todo }}</li>

</ul>

v-model

  • Binds form fields to Vue object data. The HTML value will also be pre filled with the variable value
  • Works with data and computed properties
  • You can also reference these variables to be dynamically updated elsewhere in the HTML using {{ variableName }}
  • Example:

<input v-model="username" />

        

const app = new Vue({

   el: '#app',

   data: { username: 'Jane' }

});

v-on

  • Event handling: A way to listen for DOM events and run Javascript when they are triggered
  • A shorthand for v-on is @.
  • It often accepts the name of a method you’d like to call, i.e. @click

v-on:click

  • When an element is clicked, the specified javascript method will run

<button v-on:click="tweets.push(newTweet)">Add Tweet</button>

v-bind

  • Takes a value from the data object and uses it as a prop value for the corresponding component. Recall: a prop is dy

<tweet v-for="tweet in tweets" v-bind:message="tweet"></tweet>

<!-- Inserts a tweet as the prop for the message variable in the Tweets component -->

<!-- Recall: tweets is an array of strings from the data object, which is inside the options object of the Vue constructor of (in app.js) -->

Event handling modifiers

  • Properties that can be added to directives to change their behavior
  • One use case is the functionality of Event.preventDefault() in vanilla javascript, which prevents the browser from doing its default event-handling behavior. For instance, calling it on a form submit event handler would prevent the page from automatically reloading after form submission
  • The Vue way to do it is:

<form v-on:submit.prevent="submitForm">

   <!-- form elements here -->

</form>

  • Documentation lists available modifiers

Input modifiers

  • A way to modify directives like v-model. Allows data bound to the Vue object to be processed in a particular way
  • This is especially helpful for processing form input
  • A few examples are:
  • .number - convert value in form field to a number
  • .trim - trim out whitespace from form field
  • .lazy - only update values after changes occur instead of after every keystroke
  • Additional ones in documentation

Form-Specific Syntax

Radio button bindings

  • The multiple radio button inputs get the same v-model directive and value
  • Example:

const app = new Vue({

   el: '#app',

   data: { review: '' }

});

<legend>Rate me!</legend>

<input type="radio" id="goodReview" value="good" v-model="experienceReview" />

<label for="goodReview">Good</label>

<input type="radio" id="neutralReview" value="neutral" v-model="experienceReview" />

<label for="neutralReview">Neutral</label>

<input type="radio" id="badReview" value="bad" v-model="experienceReview" />

<label for="badReview">Bad</label>

Checkbox bindings - multiple

  • Requires a dynamic array to be bound to these elements. The array will be filled with the markup value value (i.e. search)
  • Example:

const app = new Vue({

   el: '#app',

   data: { requestedFeatures: [] }

});

<legend>Which of the following features would you like to see added?</legend>

<input type="checkbox" id="search-bar" value="search" v-model="requestedFeatures">

<label for="search-bar">Search Bar</label>

<input type="checkbox" id="ads" value="ads" v-model="requestedFeatures">

<label for="ads">Ads</label>

<input type="checkbox" id="new-content" value="content" v-model="requestedFeatures">

<label for="new-content">New Content</label>

Checkbox bindings - single

  • Instead of an array like for multiple checkboxes, use a boolean
  • Example:

<legend>Would you recommend this site to a friend?</legend>

<input type="checkbox" v-model="wouldRecommend" />

Dynamic CSS Syntax

v-bind:style

  • Add a v-bind:style directive with keys consisting of CSS properties and values to be dynamic Vue variables
  • Note: hyphenated variables like font-size need to be in single quotes
  • Note: computed values can also be used for styling
  • Example using non-computed values:

<body>

   <div id="app">

       <h2 v-bind:style="{ color: breakingNewsColor, 'font-size': breakingNewsFontSize }">Breaking News</h2>

   </div>

</body>

const app = new Vue({

   el: "#app",

   data: {

       breakingNewsColor: 'red',

       breakingNewsFontSize: '32px'

   }

});

Computing style objects

  • Another design pattern: let the Vue object handle the specific styling. Pass the single object into v-bind:style
  • Benefits:
  • Makes the HTML cleaner
  • Easier to make reusable objects and not repeat the same code
  • Can store the object in a computed object that returns if a certain condition is met
  • Example: the below example has the same output as the one above

<body>

   <div id="app">

       <h2 v-bind:style="breakingNewsStyles">Breaking News</h2>

   </div>

</body>

const app = new Vue({

   el: "#app",

   data: {

       breakingNewsStyles: {

           color: 'red',

           'font-size': '32px'

       }

   }

});

Invalid email styles

<body>

   <div id="app">

       <label for="email">Email</label>

       <input type="text" id="email" v-model.trim.lazy="email" v-bind:style="[touchedEmailStyles, invalidEmailStyles]" />

   </div>

</body>

const app = new Vue({

   el: "#app",

   data: {

       email: ""

   },

   computed: {

       emailIsValid: function() {

           return this.email.includes('@');

       },

       touchedEmailStyles: function() {

           if (this.email) {

               return {

                   'border-color': '#bdbcbc',

                   'border-width': '2px'

               }

           } else {

               return {

                   'border-color': '#e0e0e0',

                   'border-width': '2px'

               }

           }

       },

       invalidEmailStyles: function() {

           if (this.email && !this.emailIsValid) {

               return {

                   'background-color': '#ffeded',

                   'border-color': '#da5252'

               }

           }

       }

   }

});
