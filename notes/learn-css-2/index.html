Flexbox terms

Flexbox

  • Tool that simplifies how to position elements, especially when the window size changes. Two important components:
  • Flex container: element on page with flex items
  • Flex item: All immediate child elements of flex container
  • Child elements change in size and location based on the size/position of their parent element
  • Any element can be a flex container
  • Some flexbox properties apply to flex containers, other to flex items

Flexbox for your entire page layout

  • Its main purpose is to lay out components in a single horizontal or vertical direction. However some developers use it for their whole page layout
  • Consider using CSS grid for whole page layout

Position of flex containers + items

  • Flex containers are block elements. No other elements go on the same line as it
  • Flex items do not begin on new lines

Main axis

  • Horizontal line from left to right

Cross axis

  • The line from the top to the bottom of a container
  • Used for align-items and align-content properties

Nested flexbox

  • Putting a flex container inside another one
  • Use when the content of a flex item needs to be easily ordered

Properties for flex containers

display: flex

  • To set an element as flex container, set the above CSS

display: inline-flex

  • If we want multiple flex containers to display inline with each other
  • Note: you need to set a width of the container or else it’ll take up the whole page
  • Recall how this is similar to display: inline
  • Example CSS:

.container {

   width: 150px;

   border: 1px solid grey;

   display: inline-flex;

  }

justify-content

  • Property that tells how to position items from left to right
  • Most common values:
  • flex-start: Puts all items in order starting from left of the parent container. No extra space is between them
  • flex-end: Position all items in order with the final item on the right side of parent container. No extra space between items
  • center: all items in order in the center of parent container. No extra space between them
  • space-around: Items positioned with equal space before an after each one
  • space-between: Items positioned with equal space between them, no extra space before the first or after the last element

align-items

  • Align flex items vertically inside a container
  • Five common values:
  • flex-start| All elements at top of parent
  • flex-end: All elements at bottom of parent
  • center: Center of all elements is halfway between top and bottom of parent
  • baseline: Bottom of all items are aligned with each other
  • stretch: Items stretch from top to bottom of the container when possible. If there is a specified height, the item will not stretch. This is the default value.

flex-wrap

  • Tell flex items to move to the next line instead of shrinking to fit the container
  • 3 values allowed:
  • wrap: child clements of flex container move down to next line
  • wrap-reverse: same as wrap but inverts the order of rows
  • nowrap: prevents items from wrapping. It’s the default value
  • Example for items that were all the same size before browser window shrunk:

align-content

  • It’s like align-items but for multiple rows: space the rows of a flex container with multiple rows
  • Common values:
  • flex-start: all rows at top of parent, no space between
  • flex-end: all rows of elements at bottom of parent, no space between
  • center: all rows of elements in center of parent element, no space between
  • space-between: all rows of elements spaced evenly from the top to bottom of container, with top and bottom elements on the very top and very bottom, respectively
  • space-around: all rows of elements spaced evenly from top to bottom, with same amount of space at both top and bottom
  • stretch: if no height specified, elements rows will stretch to fill the parent container from top to bottom. This is the default value
  • Example on the right:

flex-direction

  • Swaps the main axis and cross axis to order
  • Four values:
  • row: position items from left to right starting from top left corner (default)
  • row-reverse: position items from right to left starting with top right corner
  • column: position items from top to bottom starting with top left corner
  • column-reverse: position items from bottom to top starting from bottom left corner
  • For example, flex-direction: column; will order flex items vertically instead of horizontally
  • Example:

flex-flow

  • Declares  flex-direction and flex-wrap properties in one line. They are given in that order
  • Example:

.container {

  display: flex;

  flex-flow: column wrap;

}

Properties for flex items

flex-grow

  • Specify if items should grow to fill a container by absorbing space. Note that items do not do this by default (default value is 0). However they do shrink to fill a container
  • A value of 2 stretches 2x as much as a value of 1
  • Note: if a max-width value is set, an item won’t grow more than that

flex-shrink

  • Tells items to shrink to fill a container and in what proportions
  • Note: The default value for flex-shrink = 1
  • A value of 2 shrinks 2x as much as a value of 1
  • Note: if a max-width or min-width value is set, an item won’t grow more than that

flex-basis

  • Give the width of an item before it stretches/shrinks

flex

  • Property that tells how elements stretch and shrink all in one line. Contains flex-grow, flex-shrink, and flex-basis
  • Example: .big { flex: 2 1 150px; }
  • To set only flex-grow and flex-shrink, you can do .big { flex: 2 1 }

Grid terms

Grid

  • A tool used to lay out web pages
  • Note: Flexbox is mainly used for positioning items in 1D rows or columns; the grid helps with 2D layouts
  • To make an element a grid container, set display to one of:
  • grid: block-level grid
  • inline-grid

Default vs. Flexbox vs. Grid

default

display: flex;

display: flex;

flex-direction: column;

display: grid;

  • Code for the above:

<!DOCTYPE html>

<html>

<style>

   .container {

       border: 2px blue solid;

       height: 500px;

       width: 500px;

       /* display: */

   }

 

   .item {

       background-color: beige;

       color: black;

       border-radius: 5px;

       border: 2px dodgerblue solid;

   }

</style>

<head>

   <link rel='stylesheet' href='style.css'>

</head>

<body>

   <div class="container">

       <div class="item a">A</div>

       <div class="item b">B</div>

       <div class="item c">C</div>

       <div class="item d">D</div>

       <div class="item e">E</div>

   </div>

</body>

</html>

fr

  • A relative sizing unit that defines the size of columns/rows as a fraction of the grid’s length and width
  • I.e. grid-template: 2fr 1fr 1fr / 1fr 3fr 1fr;
  • For the rows: first row gets 2 parts of total height, second row gets one, and third row gets one
  • You can use fr with other units as well (i.e. px), and the fractions just represent available space

Grid items

  • The child elements of a grid element. Note that grid elements can take up multiple columns/rows

Grid lines

  • Unique column and row numbers in a grid
  • Values start at 1 and end at a value that’s 1 greater than number of rows/column the grid has
  • I.e. 8 rows → grid lines 1-9

Two axes in a grid layout

  • The column (aka block) axis: top to bottom
  • The row (aka inline) axis: left to right

Implicit grid

  • Algorithm built into a grid that gives the default behavior for elements when there are more than fit into the grid specified
  • Default behavior: fill up rows first, adding new rows as necessary. New rows are only tall enough to hold the content within them

Grid container properties

grid-template-columns

  • Because grids only contain one column by default, this property tells how many columns and how wide each column should be
  • Example:

.grid {

  display: grid;

  width: 500px;

  grid-template-columns: 20% 50%;

}

  • Two columns: one is 20% of grid width and one is 50%
  • You can also do px or a mix of pixels/percents
  • Note: if you use pixels and the page is too small, you can get overflow

grid-template-rows

  • Same as grid-template-columns except for rows
  • By default, rows are sized to fit evenly inside a grid. Using percentages maintains this

grid-template

  • Shorthand property that replaces grid-template-columns and grid-template-rows
  • Rows come first, then columns
  • Example:

.grid {

  display: grid;

  width: 1000px;

  height: 500px;

  grid-template: 200px 300px / 20% 10% 70%;

}

repeat()

  • A function for rows and columns that duplicates the specifications a given number of times
  • Example 1: grid-template-columns: repeat(3, 100px); gives 3 columns each 100px wide. Equivalent to 100px 100px 100px
  • Example 2: Very helpful with fr: the above could be repeat(3, 1fr)
  • Example 3: you can also do it with multiple values in second parameter:
  • grid-template-columns: repeat(2, 20px 50px)
  • This creates 4 columns: first and third are 20px, 2nd and 4th are 50px

minmax()

  • You can set min and max dimensions to rows and columns
  • Example: if an image is 100px wide, you don’t want its column to go any thinner than 100px
  • Example: in the below code, the window won't let you get smaller than the min in image on top. Fully expanded window shown below

<!DOCTYPE html>

<html>

<style>

  .grid {

      display: grid;

      border: 2px blue solid;

      height: 500px;

      grid-template: repeat(3, 1fr) / 1fr minmax(100px, 200px) 1fr;

  }

   .box {

      background-color: beige;

      color: black;

      border-radius: 5px;

      border: 2px dodgerblue solid;

  }

</style>

<head>

   <link rel='stylesheet' href='style.css'>

</head>

<body>

   <div class="grid">

       <div class="box a">A</div>

       <div class="box b">B</div>

       <div class="box c">C</div>

       <div class="box d">D</div>

       <div class="box e">E</div>

       <div class="box f">F</div>

       <div class="box f">G</div>

       <div class="box f">H</div>

       <div class="box f">I</div>

   </div>

</body>

</html>

grid-row-gap

  • Puts blank space between rows. In px

grid-column-gap

  • Puts blank space between columns

grid-gap

  • Shorthand property. grid-row-gap, grid-column-gap

grid-template-areas

  • A way to name the columns and rows of your webpage
  • This is helpful for setting them as values in the grid-row-start, grid-row-end, grid-column-start, grid-column-end, and grid-area properties
  • I.e. create 2 columns and 4 rows, named and sized as such:

.container{

   grid-template-areas: "header header"

                        "nav nav"

                        "info services"

                        "footer footer";

   grid-template-rows: 300px 120px 800px 120px;

   grid-template-columns: 1fr 3fr;

}

Cover an HTML element with a grid-template-area

  • The grid-area tells that section to cover a specific grid area from the above code

<style>

   nav {

       grid-area: nav;

   }

 

   .info {

       grid-area: info;

   }

</style>

<div class="container">

   <nav>Links!</nav>

   <section class="info">Info!</section>

</div>

grid-template

  • Shorthand for grid-template-areas. Rows first, then columns
  • I.e.

.container {

   display: grid;

   grid-template: repeat(8, 200px) / repeat(6, 100px);

 }

 

justify-items

  • Positions grid items along the row axis
  • Commonly-used values:
  • start: Align grid items to left side
  • end: Align grid items to right side
  • center: Align grid items to center of grid area
  • stretch: Stretch items to fill grid area

justify-content

  • Tells how to position a grid within its parent element along the row axis
  • Commonly-used values:
  • start
  • end
  • center: centers content horizontally
  • stretch: expands horizontally
  • space-around: equal amount of space on each side of grid element
  • space-between: equal amount of space between grid items. No space on sides
  • space-evenly: even space between grid items and on the ends

align-items

  • Positions grid items along the column axis
  • Commonly-used values: start/end/center/stretch

align-content

  • Tells how to position a grid within the rows along the column axis
  • Same values as justify-content

grid-auto-rows

  • Tells the height of rows implicitly added to grids
  • The explicit counterpart is grid-template-rows
  • Accepts same values as grid-template-rows: px, %, fr, and repeat()

grid-auto-columns

  • Tells the width of columns implicitly added to grids
  • The explicit counterpart is grid-template-columns
  • Accepts same values as grid-template-columns: px, %, fr, and repeat()

grid-auto-flow

  • Tells whether new implicit elements should be added to rows or columns first
  • Values
  • row: fill rows from left to right and then create new rows (default)
  • column: fill columns from top to bottom and then create new ones
  • dense: option to use algorithm that fills holes earlier in the grid if it needs smaller elements
  • Example: grid-auto-flow: column dense;

Grid item properties

grid-row-start / grid-row-end

  • The starting and stopping rows (grid lines) that an item will take up
  • Note: start can be greater than end, and you can have negative values
  • Example for an 8-row grid:

.grid {

   display: grid;

   border: 2px blue solid;

   height: 500px;

   width: 500px;

   grid-template: repeat(4, 1fr 2fr) / repeat(4, 3fr 2fr);

   grid-gap: 5px;

}

.a {

   grid-row-start: 5;

   grid-row-end: 7;

   /* Aka grid-row: 5 / 7; */

}

grid-row

  • Shorthand for grid-row-start, grid-row-end
  • Note: if the container is defined with a grid-gap, that will be included in the grid-row size
  • I.e. item spans 2 columns with a width of 100px each and a 10px gap, it will be 210px wide

grid-column-start / grid-column-end

  • Same as grid rows for spanning columns

grid-column

  • Same as grid-row but for spanning columns

span

  • A different way to start/end an item spanning columns or rows
  • Helpful to avoid off by one errors from miscalculating the end of a grid line
  • I.e. .item { grid-column: 4 / span 2; } equals .item { grid-column: 4 / 6; }. This takes up columns 4 through 5. Two columns total

grid-area

  • Shorthand for grid-row and grid-column
  • The format goes grid-area: grid-row-start / grid-column-start / grid-row-end / grid-column-end;
  • Example: grid-area: 2 / 3 / 4 / span 5;

Overlap grid elements

  • Use grid-area to define locations for elements; such locations can overlap
  • Remember to use a z-index to define what goes on top

justify-self

  • How an individual element should position itself within the row axis
  • Overrides justify-items
  • Accepts start/end/center/stretch

align-self

  • How an individual item should position itself within the column axis
  • Overrides align-items
  • Accepts start/end/center/stretch

Transitions terms

CSS transitions

  • Control how visual appearance changes happen: create state changes
  • 4 parts of a transition can be controlled:
  • Which CSS properties change
  • How long a transition takes
  • How much time before a transition starts
  • How a transition accel        erates
  • Properties that can be animated: https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_animated_properties

Transitions properties

transition-property

  • Tells the property that we want to transition

transition-duration

  • Tells how long the transition should take
  • Specified in s or ms
  • Default value is 0s
  • Note: a human eye blink is ~400ms

transition-timing-function

  • Describes the pace of the transition
  • Common values:
  • ease: start slow, speed up in middle, stop suddenly
  • ease-in: start slow, accelerate quickly, stop suddenly
  • ease-out: start suddenly, slow down, end slowly
  • ease-in-out: start slow, get fast in middle, end slowly
  • linear: constant speed

transition-delay

  • The amount of time to wait before starting a transition
  • Default value is 0s
  • Can be in ms or s

transition

  • Shorthand for the four CSS transition properties
  • Order: transition-property, transition-duration, transition-timing-function, transition-delay
  • If you leave out a property, it will be set as default. However you must define duration if you want delay. Otherwise the browser doesn’t know to do delay
  • You can also transition multiple properties at once with commas
  • Examples:
  • transition: color 1.5s linear 0.5s;
  • transition: color 1s linear, font-size 750ms ease-in 100ms;

all

  • Value for transition that applies the transition values to all properties, saving the need to write multiple copies of the same transition properties
  • I.e. transition: all 1.5s linear 0.5s;

Relative measurements

Responsive design

  • The ability for a website to resize and reorganize itself based on the size of other stuff on the site and the size of the screen the site is being viewed on

Relative measurements

  • %, em, and Fr values instead of px. Preferred for responsive websites
  • This makes sure parts of the site stay intact regardless of screen size or layout

Percentages

  • Can be used to size non-text HTML elements
  • Sizing is done relative to their parent elements for height and width
  • Sizing is done relative to the width of the parent only for padding and margin
  • Commonly used for box-model values: width, height, padding, border, and margins
  • Commonly used for positioning properties: top, bottom, left, right
  • Note: because the box model includes padding, borders, and margins, setting an element width to 100% can cause overflow.
  • Only use 100% when there’s no padding, border, or margin

Limiting relative sizes of elements

  • If a window becomes too big or too small, an element can lose its integrity
  • Use min-width, max-width, min-height, and max-height with px values, to combat this

Best practices for images/videos relative sizing

  • Set overflow: hidden for the CSS code of the container to make sure anything with dimensions larger than the container will be hidden
  • Set height: auto for images so they automatically scale proportionally with window width
  • Set display: block so that images aren’t on an inline-block display. This will prevent them from trying to align with other content like text
  • Recall from Learn CSS I: you can use background-size: cover to scale a background image proportionally
  • Common design pattern for image/video scaling:

.container {

   width: 50%;

   height: 200px;

   overflow: hidden;

 }

 

 .container img {

   max-width: 100%;

   height: auto;

   display: block;

 }

Relative measurements - fonts

Em

  • The font size of the current element or the default font size set by the browser if no current element font is given
  • I.e. base font of browser 16 px → 1em = 16px, 2em = 32px, etc.
  • Historically it represented the width of a capital M in the font and size used; this is no longer true

How to define a base font size for the element

  • Use font-size
  • For instance in below, base font size becomes 18px. Now 1.5em equals 27px

.my-class {

   font-size: 18px;

 }

.my-class h1{

   font-size: 1.5em;

}

Rem

  • Root em - like em, but it checks the root (the <html> tag) instead of parent element
  • For most browsers, default size of <html> is 16px
  • Syntax example: 3.75rem
  • Change root by modifying: html { font-size: 20px; }

Responsive web design terms

Responsive web design

  • When a website responds to the size of a screen it’s being viewed on, and adapt accordingly so it can be used easily

Media queries

  • When CSS detects the size of the current screen and then applies different styles depending on screen width

Media features

  • Conditions that must be met in order to render a particular media query
  • Tells the CSS compiler to apply styles to devices with these conditions, i.e. pixel width or screen resolution

CSS rule

  • Just like in Learn CSS I, these are the property value pairs assigned to elements. For responsive web design, they go within the media query

Dots per inch (DPI)

  • Measure of screen resolution: # of individual dots that can be put in a span of 1 inch
  • For example: we only want to give higher quality images or videos to users with screens that can support them

Comma separated list

  • Can be used instead of the and keyword if only one of multiple media features must be met
  • Example:

@media only screen and (min-width: 480px), (orientation: landscape) { }

Breakpoints

  • The points at which media queries are set
  • As such, these are often the screen sizes where your web page doesn’t appear properly. So you need to fix it using a breakpoint
  • Note: you could theoretically have infinite breakpoints. New devices are also released every year. Instead, best practice is to resize browser and look where the layout breaks or becomes weird. Those are your breakpoints - they should cover most devices

Responsive web design syntax

@media

  • Keyword to begin a media query rule
  • Can include the following operators and features:
  • only
  • Media type i.e. screen
  • and
  • Media feature i.e. (max-width: 480px)
  • Example:

@media only screen and (max-width: 480px) {

 .page-title {

   width: 270px;

 }

}

only

  • Media query operator that applies the rule(s) to only one media type. i.e. screen

media type

  • I.e. screen. Applies to all devices that satisfy that media type

and

  • Operator used to require multiple media features and types

min-width / max-width

  • Set screen size as part of the media query
  • Examples: note they are equivalent and are both valid design patterns

@media only screen and (min-width: 320px) and (max-width: 480px) { }

@media only screen and (min-width: 320px) {

   /* ruleset for >= 320px */

}

@media only screen and (min-width: 480px) {

   /* ruleset for >= 480px */

}

min-height / max-height

  • Media features height of screen size in media query

min-resolution / max-resolution

  • Media features that target resolution
  • In dpi or dpc
  • Example: the below adds a higher resolution logo for high resolution screens

@media only screen and (min-resolution: 150dpi){

   .logo {

     background-image: url("../img/spaceship@2x.png");

   }

 }

orientation

  • Media feature for portrait or landscape

print

  • Media type that dictates properties if a page is printed
  • This helps reduce clutter on the page - i.e. ads, comments, or changing href links so users still understand it’s a link:

@media print {

   a[href^="http"]:after {

     content: " (" attr(href) ")";

   }

 }

Variables terms

CSS Variables

  • Allow the same information to be maintained and referenced in different locations
  • Also makes code easier to read
  • Declared in CSS using --

Custom properties

  • CSS’s own variables that hold specific colors, i.e. --main-red-color instead of #7f3f3f

Scope

  • Where a variable will work based on where it’s declared
  • 2 types: local and global
  • Local only affects the HTML element it is declared, along with any children
  • Global: applies everywhere in the html file
  • Note: it’s common practice to define all variables globally
  • Times you might prefer local: huge website so lots of them don’t pile up in :root

CSS Variable Inheritance

  • If a variable is not explicitly defined on the child element, the parent variable is automatically used if there is one (it inherits)
  • Recall: not all property values are inherited by children. For example widths, margins, paddings, and borders

Overriding variables

  • You can do this in CSS - i.e. variable is declared globally but you want something else for local elements

Variables syntax

Define a local variable

  • See example below for syntax for locally scoped variables
  • Note: defining a variable does not immediately do anything. You have to use it
  • Conventions:
  • They’re case sensitive. Keep things lowercase
  • Use hyphen-delimited-strings instead of CamelCase

h1 {

   --main-header-color : #DADECC;

 }

var( )

  • How to use a variable
  • Note: you can also define variables in place of other variables. This can help keep things modular and make code more readable
  • Note: if you try to overwrite a redefined variable after it’s been defined, it won’t change
  • Examples:

h1 {

   --main-background-color: #DADECC;

   background-color: var(--main-background-color);

 }

html {

   --custom-purple: #FF64ED;

   --main-color: var(--custom-purple);

 }

 

 body {

   /* Trying to override the --custom-purple variable defined in the html selector rules */

   --custom-purple: #FFFFF;

   background: var(--main-color);

   /* Element’s background color will still be #FF64ED */

 }

Define a global variable

  • Define them in the :root pseudo-class
  • Typically :root refers to the <html> element
  • Example:

:root {

   --menu-color-blue: blue;

 }

Fallback values

  • A second optional argument for a variable (in var()) in case the first one fails
  • Another variable can also be used
  • Examples: if the variable hasn’t been defined or returns an invalid value, the fallback is used
  • Note: do not use fallback values to fix browser compatibility problems
  • Note: internet explorer does not support fallback values

body {

   background: var(--main-background-color, #F3F3F3);

   color: var(--main-color, var(--favorite-orange, red));

 }

Variables used with media queries

  • Incredibly powerful as websites scale up-you don’t have to set new values in every media query
  • Example: use a dark theme if screen is below a certain size

@media screen and (min-width: 600px) {

   :root {

     /* Light Color Theme */

     --body-background: lightblue;

     --inner-margin: 6px;

     --body-text-color: black;

     --font-size: 18px;

   }

 }

 

 @media screen and (max-width: 600px) {

   :root {

     /* Dark Color Theme */

     --body-background: #000;

     --inner-margin: 12px;

     --body-text-color: #fff;

     --font-size: 12px;

   }

 }

Functions

Functions

  • Blocks of organized, reusable code that do a single and related action
  • They are helpful in keeping code modular and reusable

Functions in CSS

  • A CSS value that’s used in place of a hardcoded value for a property
  • Some functions can take multiple arguments
  • Note: functions in CSS are more limited than most languages. We can’t create our own but can access predefined ones including var( ) and repeat( )
  • List of all functions is at https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Functions
  • Example syntax:

h1 {

   property: function-name(argument);

 }

url()

  • Link local or external resources, i.e. background: url('images/my-image.jpg'); 
  • You might still need to use properties after the url like background-repeat: no-repeat;

calc()

  • Calculates a mathematical expression
  • Cal also take in mixes of units, i.e. margin: 15px calc(1.5vw + 5px);
  • There must be white space between the operators and numbers
  • The function can also be used as a value in another function
  • + and - : both numbers must have units given
  • / : divisor must be a unitless number
  • * : one of the two operands must be unitless

min() / max()

  • Select the smallest or largest value, respectively, from a range of values
  • This can be helpful for responsive elements
  • Example: the below two rules are the same

.content {

   width: 50vw;

   max-width: 500px;

 }

.content {

   width: min(500px, 50vw);

 }

clamp()

  • Keeps a value to be within an upper and lower bound
  • Requires 3 parameters: minimum value, preferred value, maximum value
  • Preferred value is kept unless the min or max conditions are satisfied
  • Example:

.main-text{

   font-size: clamp(12px, 1.5vw, 48px);

 }

brightness()

  • Takes a single value for the amount - in 0-1 or 0%-100%
  • Is a function for filter and backdrop-filter

blur()

  • Applies a Gaussian blur to the element. 1 argument: the radius of the blur as a length (with units)

drop-shadow()

  • Creates an element that looks like a shadow of an element
  • Format: drop-shadow(offset-x offset-y blur-radius color)
  • offset-x and offset-y are horizontal and vertical offset, respectively
  • blur-radius says how blurred the shadow is
  • color = shadow’s color

saturate()

  • Modifies the saturation of an image

transform

  • A property that lets any HTML element be changed - there are then associated CSS functions that scale, rotate, and distort
  • Can be 2D and 3D transformations

scale()

  • Transform function that resizes an element on a 2D plane.
  • 1 or 2 arguments (x and y). If only one argument given, scales proportionally in both directions
  • You can also use scaleX() and scaleY()

rotate()

  • Rotate an element around its origin on the 2D plane
  • Takes in a single argument - an angle (must be given with deg)
  • Positive angle is clockwise, negative is counter clockwise
  • You can also rotate around other points - see docs

translate()

  • Move an element from its position to another position
  • Can take 1 or 2 arguments (x direction and y direction). If only 1 argument is given, translate will only be along x axis
  • Example:

.shifted {

   transform: translate(0px, 100px);

 }

Appearance terms and best practices

font-size

  • Should be at least 18 pixels, especially for smaller screens

line-height

  • Minimum of 1.5 within paragraphs
  • You don’t want it too large either or lines will appear too far together

letter-spacing

  • Typically keep with the default normal value

text-align

  • Use left, right, or center
  • Justify can decrease readability due to uneven word spacing - like a newspaper

Paragraph width

  • Recommended 45-85 characters per line, 65 is ideal
  • Remember - you can use ch instead of px

Contrast ratio

  • The difference between two colors - as per their distance apart on the color wheel
  • 1:1 (both colors are the same) to 21:1 (the two colors are black and white)

Color contrast recommendations

  • The Web Content Accessibility Guidelines (WCAG) has 3 levels of criteria - A through AAA. A is the minimum
  • WCAG rules for contrast ratio:
  • 4.5:1 (AA criteria)—Text that is less than 24px (not bold) and 19px (bold).
  • 3:1 (AA criteria)—Text larger than 24px (not bold) and 19px (bold).
  • 7:1 (AAA criteria)—Text that is less than 24px (not bold) and 19px (bold).
  • 4.5:1 (AAA criteria)—Text larger than 24px (not bold) and 19px (bold).

<abbr>

  • Use this HTML element for a user to get the full name of an abbreviation if they hover over it
  • I.e. <abbr title='Cascading Style Sheets'>CSS</abbr>

Links

  • Underline links using the text-decoration property

cursor

  • Use cursor: pointer; for elements like buttons - so users know they’re interactive

:visited

  • Let users know they’ve visited a link from your page previously
  • Example:

a:visited { color: purple; }

:focus

  • Let users know where they are on the page with additional styling
  • This helps, especially when users are using other navigation besides a mouse (i.e. keyboard)
  • Example:

input:focus { border-color: blue; }

  • You can also use this pseudo class for form validation - i.e input is an error so make it red, etc:

Designing for screen readers

  • Screen readers process in order of the DOM
  • It can be possible to have a mismatch between what’s being represented and what’s on the page (i.e. elements being read out of order)

Hide from screen readers but display visually

  • Recall from Learn CSS I: you can hide content visually. This hides content from both screen readers and visual
  • If you want to hide from  screen readers but display visually, set <p aria-hidden=”true”> as a property in the html
  • Note: this can be for any element, not just text

Vendor prefixes

  • Property names to test new CSS features before browsers completely supported them
  • Are not intended to be used in production websites, but sometimes they’re needed to use cutting-edge features (i.e. in Chrome Canary) or to support older versions of browsers
  • When these are adopted into all browsers, the prefix is no longer needed
  • Prefixes:
  • -webkit- : Chrome, safari, new Opera versions, iOS browsers
  • -moz- : Firefox
  • -ms- : IE and Edge
  • -o- : Old pre-WebKit Opera versions
  • The below tool can automatically include all vendor prefixes for features that need them
  • https://github.com/postcss/autoprefixer
  • Example: the below code doesn’t add the feature to chrome unless the vendor prefix line is included

strong{

 background-clip: text;

 -webkit-background-clip: text;

}

Polyfills

  • Javascript code tools that rewrites HTML and CSS codes to simulate features not currently supported in browsers
  • Tools:
  • https://github.com/Modernizr/Modernizr
  • https://github.com/financial-times/polyfill-service
  • Example use in html document:

<script src="https://polyfill.io/v3/polyfill.min.js"></script>
