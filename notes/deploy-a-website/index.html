General Terminology

Static web page

  • A page delivered to a browser exactly as stored, displaying the same information for all users

Dynamic web page

  • Pages which are generated by a web application

View site locally

  • View a site on your own computer only. Not on the internet. Your computer is acting as the hosting server

GitHub Pages

  • Public web pages that are hosted and published through GitHub
  • Offers integration and support for Jekyll
  • Note: repos must be in a naming format of your-user-name.github.io

Domain name

  • Human friendly names that link to servers on the internet

Web hosting server

  • Stores your website files in a location for others to get them
  • Servers owned by web hosting companies like Namecheap
  • I.e. Namecheap’s “Stellar” hosting service

Domain name server

  • Stores relationships between IP addresses and domain names for use with a browser

TLD

  • Top level domain: .com, .org, .ly, etc.

Second level domain

  • Newer website domains such as .tech, .link, .website, etc.
  • These have no overall impact on search engine performance

Domain Name System (DNS)

  • A global system that stores which domain names correspond to which servers

Hosted Zone

  • A group of DNS records for a single domain

Name Server (NS)

  • Associates domain names with proper DNS records, and gives the owner exclusive control over DNS records        
  • When you enter a domain name, the name server is the first step and they also give the browser other info about the website

A record

  • Address record, which directs a domain name to an IP
  • Also known as Alias record
  • For example, associate new custom domain name with Github’s servers

Subdomains

  • Domains that are part of the root domain. I.e. https://www.yourcustomdomain.com/ is a subdomain of https://yourcustomdomain.com/

CNAME

  • Canonical Name, which is used for setting up subdomains by giving a domain name to be used as an alias

HTTPS

  • Adds a layer of encryption to prevent people to snoop on or tamper with website traffic

Mixed content

  • If you enable HTTPS but the HTML on your site references images, CSS, or JS over HTTP. Convert all http to https
  • Serving mixed content makes your site less secure

Jekyll Terminology

(note: I did not end up using Jekyll)

Jekyll

  • Tool that provides simple static sites
  • It’s a common way to generate ready-to-public websites
  • Note: you can expand off of it with your own content - just have the HTML you want in index.html

new

  • Create a directory to fill with Jekyll’s content

serve

  • Starts a local server to serve the files to your computer
  • Helps with previewing site changes as well

Process to create website with GitHub pages/AWS

  1. Buy domain on AWS Route 53
  2. Commit a CNAME file to GitHub as your AWS-purchased domain name to be the subdomain
  3. Create DNS records to map domain names to servers:
  1. AWS Route 53 → Hosted Zones → Click on your domain name
  2. Confirm name server record
  3. Click “Create record”
  1. Leave “Name:” blank
  2. Set “Type:” to be A - IPv4 address
  3. In “Value” enter the following GitHub-owned IP addresses:

192.30.252.153

192.30.252.154

  1. Click “Create records”
  1. Create a CNAME record for www.:
  1. Click “Create record” on hosted zone
  2. Set “Name:” to be www.
  3. Set “Type:” to be CNAME - Canonical name to set up the subdomain
  4. Set “Value” to your-user-name.github.io
  5. Click “Create records”
