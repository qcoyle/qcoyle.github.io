Terminology

Command line

  • A text interface for the computer
  • Takes in commands and passes them to the OS to run

Root directory

  • First directory in a filesystem
  • It’s the parent of all other directories/files in the filesystem
  • Going “down” a directory means going into children
  • Note: ~ represents your home directory (i.e. cd ~)
  • Example to reference a file from the root directory:

~/git/scripts/nodejs$ cp index.html ~/Downloads/index.html

Command

  • An instruction to computer to perform a specific task

Shell prompt

  • The $ in the terminal

Options

  • A way to modify the behavior of commands, usually with a - after the command

Wildcards

  • In addition to using filenames for arguments, can also use wildcards, *, to get groups of files
  • Examples:
  • cp * myDirectory/
  • Copy all files in current working directory into another directory
  • cp w*.txt myDirectory/
  • Take all files in working directory starting with “w” and ending with “.txt” and copy them to directory

Redirection

  • Direct the standard input, output, and error of a command to/from other files and programs

Standard input (stdin)

  • What you tell a command to do, i.e. the “hello world” in $echo “hello world”

Standard output (stdout)

  • Information output after a process is run

Standard error (stderr)

  • Error message output by a failed output

Environment

  • The session of terminal  that we are in - with particular settings and preferences which can help support commands and programming we are doing

Terminal text editor

  • Programs like Nano and Vim
  • Accepts only keyboard input

Bash profile

  • File used to store environment settings for your terminal
  • Can be found in ~/.bash_profile

Environment variable

  • Variables that can be used across commands and programs
  • Denoted in bash profile by USER=” “
  • See export in environment commands

bin directory

  • Where many terminal commands are stored
  • For example, when you type in pwd the script executed is /bin/pwd

Navigation commands

ls

  • Lists all files/directories inside the directory you are in
  • Options:
  • -a: List all contents including hidden files/directories
  • -l: List all contents in long format:
  • Access rights
  • Number of hard links (# child directories and files)
  • Username of file’s owner
  • Name of group that owns the file
  • Size of file in bytes
  • Date and time the file was last modified
  • Name of directory or file
  • -t: List contents in order of last modified
  • -alt: use multiple options together

pwd

  • Print working directory - the directory you are currently in

cd

  • Change directory - switches you into directory you specify
  • Additionally:
  • cd dir1/dir2/etc
  • cd ..
  • cd ../dir2
  • Navigate up a directory and then back down again

mkdir my_folder

  • Make a directory
  • Can also do mkdir parent/child

touch

  • Create a new file

clear

  • Clear your terminal. Doesn’t change anything

wc

  • Returns lines, words, and characters of a file

open

  • Open a file
  • open . to open current directory in finder

source

  • Read and execute commands
  • It also activates changes in the bash_profile, instead of needing to start a new terminal session

history

  • See a history of terminal commands that were output

Manipulation Commands

cat

  • Output the contents of a specified file - i.e. all the text

cp

  • Copy files or directories
  • Examples:
  • Move the contents of source.txt to destination.txt: cp source.txt destination.txt
  • Note: it will overwrite what’s in destination.txt
  • Copy a file to a destination directory: cp source.txt myDirectory
  • Copy multiple files to a destination directory: cp fil1.txt file2.txt myDirectory/

mv

  • Move files without making a copy. Can also rename file names
  • Examples:
  • mv my_file.txt myDirectory
  • mv oldFile newFile
  • Very similar to cp command

rm

  • Delete files and directories. Note that it deletes them permanently - no trash
  • Options:
  • -r to delete a directory and all of its child directories + files. Very powerful
  • Examples
  • rm removeDis.txt

Redirection Commands

echo

  • Give text input back to the terminal or redirect it somewhere

>

  • Command that redirects standard output to a file
  • Examples:
  • $ echo “Hello” > hello.txt
  • $ cat deserts.txt > forests.txt overwrites everything from deserts and puts it into forests 

>>

  • Append standard output of left file to file on the right
  • Examples:
  • $ cat deserts.txt >> forests.txt
  • $ echo 'hello world!' >> my_file.txt appends text to the file

<

  • Takes standard input from file on right and inputs it to program on left
  • Example:
  • cat < lakes.txt

|

  • Known as a pipe, it takes standard output from left command and pipes it as a standard input to the command on right
  • Example:
  • $ cat volcanoes.txt | wc | cat > islands.txt takes lines/words/characters from volcanoes.txt and then redirects that output to islands.txt. Now cat islands.txt returns 17      26     204

sort

  • Orders the contents in the standard input into alphabetical order. Note the command itself doesn’t change the file
  • To make an alphabetical file, do:
  • $ cat original.txt | sort > new.txt

uniq

  • Unique - filters out adjacent duplicate lines in a file
  • Note: if the lines are not adjacent, it will not filter them. If you want to do that, first run a sort. For example,
  • sort deserts.txt | uniq > uniq-deserts.txt

grep

  • Global regular expression print: search specified files for lines that match a pattern and return the results
  • Case sensitivity:
  • grep is case sensitive
  • grep -i lets the command be case insensitive
  • Searching directories:
  • grep just searches files
  • grep -R searches all files in a directory and outputs the filenames + lines that match results (it gives you the test inside them too)
  • -R means recursive
  • grep -Rl searches all files in a directory and outputs only the filenames with matched results.
  • You can also use regular expressions to search for patterns
  • Examples:
  • grep -i America continents.txt
  • grep -R pandas imessage-data/

sed

  • Stream editor - it accepts standard input and modifies it based on an expression before then displaying as output data
  • sed expressions:
  • s: substitution. It replaces first argument with second argument. It only does this once, and just outputs it to the command line
  • Example: sed 's/snow/rain/' forests.txt replaces the first “snow” in forests.txt with “rain”
  • g: global. Does it to all instances
  • Example: sed 's/snow/rain/g' forests.txt replaces all instances of “snow” with “rain”
  • -i: rewrites the original file
  • Example: sed -i 's/snow/rain/g' forests.txt replaces all instances of “snow” with “rain” in forests.txt, and then overwrites forests.txt with those changes

Environment Terms

These need to be done in the .bash_profile

alias

  • Create keyboard shortcuts for commonly used command
  • You can add as many aliases as you want
  • Example: 
  • alias pd="pwd" makes pd the pwd command for your terminal

export

  • Makes a variable available to all child sessions from the terminal session you are in - keeps a variable across programs
  • Note: always use $ when returning an environment variable’s value
  • Example:
  • export USER = “jane” && echo $USER

PS1

  • Environment variable that defines the makeup and style of the command prompt

$HOME

  • Environment variable that displays path of home directory

$PATH

  • Environment variable that stores a list of directories separated by a colon which are the scripts for the command line to execute

env

  • Returns a list of environment variables
  • Manipulation example:
  • env | grep PATH shows all the values of the PATH environment variable
